# RH0438 - Element return value documentation must have text

| Property | Value |
|----------|-------|
| **ID** | RH0438 |
| **Category** | Documentation |
| **Severity** | Warning |
| **Code Fix** | ❌ |

## Description

This rule requires the `<returns>` tag to contain descriptive text.

## Why is this a problem?

An empty `<returns>` tag looks like finished documentation while still leaving the return contract unexplained.

## How to fix it

Write a short description of what the method or delegate returns.

## Examples

### Violation

```cs
/// <summary>
/// Gets the customer count
/// </summary>
/// <returns></returns>
internal int GetCustomerCount()
{
    return 0;
}
```

### Correction

```cs
/// <summary>
/// Gets the customer count
/// </summary>
/// <returns>The number of customers currently loaded</returns>
internal int GetCustomerCount()
{
    return 0;
}
```
