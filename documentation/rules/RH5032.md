# RH5032 — End-region directives must be surrounded by blank lines

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH5032       |
| **Category** | Layout |
| **Severity** | Warning      |
| **Code Fix** | ✓            |

## Description

An `#endregion` directive must have a blank line before and after it, so the directive is visually separated from the surrounding members.

## Why is this a problem?

Without the surrounding blank lines, the `#endregion` marker merges with the adjacent member and the end of the grouping is harder to see at a glance.

## How to fix it

Add a blank line before and after the `#endregion` directive. The only exception is when the directive is directly followed by a closing brace: no blank line is added between the `#endregion` and the `}`, keeping the rule consistent with the brace blank-line rules. The code fix inserts the missing blank lines automatically.

## Examples

### Violation

```cs
internal class TestClass
{
    private int _b;
    #endregion // Helpers
    private int _c;
}
```

### Correction

```cs
internal class TestClass
{
    private int _b;

    #endregion // Helpers

    private int _c;
}
```
