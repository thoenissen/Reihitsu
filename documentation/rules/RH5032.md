# RH5032 — Region directives should be followed by a blank line

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH5032       |
| **Category** | Layout |
| **Severity** | Warning      |
| **Code Fix** | ✓            |

## Description

A `#region` or `#endregion` directive should have a blank line after it, so the directive is visually separated from the following code.

## Why is this a problem?

Without a blank line behind it, a region marker merges with the following member and the grouping is harder to see at a glance.

## How to fix it

Add a blank line after the directive. The only exception is when the directive is directly followed by a closing brace: no blank line is required between the directive and the `}`, keeping the rule consistent with RH5024. The code fix inserts the missing blank line automatically.

## Examples

### Violation

```cs
internal class TestClass
{
    private int _a;

    #region Helpers
    private int _b;
}
```

### Correction

```cs
internal class TestClass
{
    private int _a;

    #region Helpers

    private int _b;
}
```
