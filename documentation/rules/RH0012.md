# RH0012 — Do not prefix local members with `this.`

| Property | Value |
|----------|-------|
| **ID** | RH0012 |
| **Category** | Clarity |
| **Severity** | Warning |
| **Code Fix** | ✓ |

## Description

This rule removes unnecessary `this.` qualifiers from member access when they are not needed for disambiguation.

## Why is this a problem?

Redundant `this.` adds noise and makes member access look more significant than it is. The qualifier should be reserved for the rare cases where it prevents ambiguity.

## How to fix it

Remove `this.` when the member can be referenced directly.

## Examples

### Violation

```cs
public int GetValue()
{
    return this._value;
}
```

### Correction

```cs
public int GetValue()
{
    return _value;
}
```
