# RH0008 — Do not use default value type constructor

| Property | Value |
|----------|-------|
| **ID** | RH0008 |
| **Category** | Clarity |
| **Severity** | Warning |
| **Code Fix** | ✓ |

## Description

This rule discourages creating a value type with an empty constructor call such as `new Guid()` when the intent is just the default value.

## Why is this a problem?

`new T()` looks like a normal constructor call even though it only produces the default value for the struct. Using `default` makes that intent clear.

## How to fix it

Replace `new T()` with `default(T)` or `default` when appropriate.

## Examples

### Violation

```cs
var id = new Struct();
```

### Correction

```cs
var id = default(Struct);
```
