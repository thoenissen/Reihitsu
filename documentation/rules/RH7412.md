# RH7412 — Interface implementation indexers should be grouped by interface regions

| Property | Value |
|----------|-------|
| **ID** | RH7412 |
| **Category** | Organization |
| **Severity** | Warning |
| **Code Fix** | ❌ |

## Description

This rule requires each indexer that implements an interface indexer to appear inside a top-level `#region` whose label matches the interface that declares the member.

## Why is this a problem?

Members implementing an interface are easier to scan when they are grouped by the interface they belong to. Generic region names such as `Indexers` hide that relationship and make types that implement several interfaces harder to navigate.

## How to fix it

Move each interface implementation indexer into a top-level region named after the implemented interface. Use the interface name only, such as `IReadOnlyList`. This applies to both explicit and implicit implementations.

## Examples

### Violation

```cs
internal interface IIndexable
{
    int this[int index] { get; }
}

internal class TestClass : IIndexable
{
    #region Indexers

    public int this[int index] => index;

    #endregion // Indexers
}
```

### Correction

```cs
internal interface IIndexable
{
    int this[int index] { get; }
}

internal class TestClass : IIndexable
{
    #region IIndexable

    public int this[int index] => index;

    #endregion // IIndexable
}
```
