# RH5201 — Method chains should be aligned

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH5201       |
| **Category** | Layout |
| **Severity** | Warning      |
| **Code Fix** | ✓            |

## Description

This rule checks that multi-line method chains have their access operators aligned with the first invoked chain link. When a method chain spans multiple lines, each continuation line's `.` or `?.` should be vertically aligned for consistent readability.

## Why is this a problem?

Misaligned method chains make it difficult to follow the sequence of operations, especially when chains are long. Consistent dot alignment provides a clear visual guide through the chain of calls.

## How to fix it

Align the leading `.` or `?.` on each continuation line with the access operator on the first line of the chain. The code fix will automatically reformat the method chain.

## Examples

### Violation

```cs
var result = list.Where(x => x > 0)
.Select(x => x * 2)
.ToList();
```

### Correction

```cs
var result = list.Where(x => x > 0)
                 .Select(x => x * 2)
                 .ToList();
```
