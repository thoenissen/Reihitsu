# RH5602 — Code must not contain trailing whitespace

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH5602       |
| **Category** | Layout |
| **Severity** | Warning      |
| **Code Fix** | ✓            |

## Description

This rule checks whether a line ends with spaces or tabs before the line break. Trailing whitespace creates noisy diffs and makes formatting less predictable. Trailing whitespace inside string, interpolated-string, and raw-string literals is treated as semantic content and is left untouched. Trailing whitespace inside comments (`//`, `/* */`, `///`, `/** */`) and inside preprocessor-disabled text (for example the body of an `#if false` block) is exempt for the same reason as tabs are for RH5601: that content may be semantically meaningful and the formatter never rewrites it.

## Why is this a problem?

Trailing whitespace does not affect execution, but it increases visual noise and causes avoidable diff churn.

## How to fix it

Remove the trailing whitespace at the end of the line.

## Examples

### Violation

```cs
internal class TestClass   
{
    void Method()␠␠␠
    {
    }
}
```

### Correction

```cs
internal class TestClass   
{
    void Method()
    {
    }
}
```
