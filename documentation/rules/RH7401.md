# RH7401 — Override methods should be grouped by base type regions

| Property | Value |
|----------|-------|
| **ID** | RH7401 |
| **Category** | Organization |
| **Severity** | Warning |
| **Code Fix** | ❌ |

## Description

This rule requires each `override` method to appear inside a top-level `#region` whose label matches the type that originally introduced the overridden member.

## Why is this a problem?

Override methods are easier to scan when they are grouped by the type that originally introduced them. Generic region names such as `Methods` hide that relationship and make inheritance-heavy types harder to navigate.

## How to fix it

Move each `override` method into a top-level region named after the type where the overridden member was first declared. Use the type name only, such as `BaseProcessor`.

## Examples

### Violation

```cs
internal abstract class BaseProcessor
{
    public virtual void Execute()
    {
    }
}

internal class DerivedProcessor : BaseProcessor
{
    #region Methods

    public override void Execute()
    {
    }

    #endregion // Methods
}
```

### Correction

```cs
internal abstract class BaseProcessor
{
    public virtual void Execute()
    {
    }
}

internal class DerivedProcessor : BaseProcessor
{
    #region BaseProcessor

    public override void Execute()
    {
    }

    #endregion // BaseProcessor
}
```
