# RH0207 — Event names should be in PascalCase

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH0207       |
| **Category** | Naming       |
| **Severity** | Warning      |
| **Code Fix** | ✓            |

## Description

Checks that event names follow PascalCase naming conventions. Each word in the name should start with an uppercase letter, with no underscores or other separators.

## Why is this a problem?

PascalCase for public members is a fundamental C# naming convention. Inconsistent event naming reduces readability and breaks expectations for consumers of the API.

## How to fix it

Rename the event to use PascalCase.

## Examples

### Violation

```cs
public event EventHandler myEvent;
```

### Correction

```cs
public event EventHandler MyEvent;
```
