# RH7410 — Interface implementation properties should be grouped by interface regions

| Property | Value |
|----------|-------|
| **ID** | RH7410 |
| **Category** | Organization |
| **Severity** | Warning |
| **Code Fix** | ❌ |

## Description

This rule requires each property that implements an interface property to appear inside a top-level `#region` whose label matches the interface that declares the member.

## Why is this a problem?

Members implementing an interface are easier to scan when they are grouped by the interface they belong to. Generic region names such as `Properties` hide that relationship and make types that implement several interfaces harder to navigate.

## How to fix it

Move each interface implementation property into a top-level region named after the implemented interface. Use the interface name only, such as `INotifyDataErrorInfo`. This applies to both explicit and implicit implementations.

## Examples

### Violation

```cs
internal interface IIdentifiable
{
    int Id { get; }
}

internal class TestClass : IIdentifiable
{
    #region Properties

    public int Id { get; }

    #endregion // Properties
}
```

### Correction

```cs
internal interface IIdentifiable
{
    int Id { get; }
}

internal class TestClass : IIdentifiable
{
    #region IIdentifiable

    public int Id { get; }

    #endregion // IIdentifiable
}
```
