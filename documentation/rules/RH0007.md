# RH0007 — Use shorthand for nullable types

| Property | Value |
|----------|-------|
| **ID** | RH0007 |
| **Category** | Clarity |
| **Severity** | Warning |
| **Code Fix** | ✓ |

## Description

This rule prefers the C# nullable shorthand `T?` over `Nullable<T>` or `System.Nullable<T>`.

## Why is this a problem?

The shorthand is the standard C# notation for nullable value types. It is shorter, easier to read, and more idiomatic.

## How to fix it

Replace `Nullable<T>` with `T?`.

## Examples

### Violation

```cs
private Nullable<int> _count;
```

### Correction

```cs
private int? _count;
```
