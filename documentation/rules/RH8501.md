# RH8501 — Code analysis suppressions must have a justification

| Property | Value |
|----------|-------|
| **ID** | RH8501 |
| **Category** | Documentation |
| **Severity** | Warning |
| **Code Fix** | ❌ |

## Description

Every `SuppressMessage` attribute must include a non-empty `Justification` value.

## Why is this a problem?

Suppressions without an explanation hide technical debt and make it difficult to understand whether the suppression is still valid.

## How to fix it

Add a meaningful `Justification = "..."` named argument to the suppression.

## Examples

### Violation

```cs
[System.Diagnostics.CodeAnalysis.SuppressMessage("Design", "CA1062")]
internal class TestClass { }
```

### Correction

```cs
[System.Diagnostics.CodeAnalysis.SuppressMessage("Design", "CA1062", Justification = "Parameter is validated by the caller.")]
internal class TestClass { }
```
