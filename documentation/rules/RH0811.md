# RH0811 — File-scoped namespaces should be used

| Property | Value |
|----------|-------|
| **ID** | RH0811 |
| **Category** | Formatting |
| **Severity** | Warning |
| **Code Fix** | ✗ |

## Description

This rule requires eligible namespace declarations to use the file-scoped form instead of the block-scoped form.

## Why is this a problem?

Block-scoped namespaces add an extra indentation level around the entire file. That makes the file longer and pushes all members further to the right without adding useful structure.

Using the file-scoped form keeps the namespace visible while reducing nesting and making the file easier to scan.

## How to fix it

Replace the block-scoped namespace with a file-scoped namespace and remove the extra indentation from the namespace contents.

## Examples

### Violation

```cs
namespace Sample.Project
{
    using System;

    internal class Example
    {
    }
}
```

### Correction

```cs
namespace Sample.Project;

using System;

internal class Example
{
}
```
