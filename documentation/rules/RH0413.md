# RH0413 - Private enums must be documented

| Property | Value |
|----------|-------|
| **ID** | RH0413 |
| **Category** | Documentation |
| **Severity** | Warning |
| **Code Fix** | ❌ |

## Description

This rule requires private enum declarations to have XML documentation. A declaration counts as documented when it has a &lt;summary&gt; tag or uses &lt;inheritdoc/&gt;.

## Why is this a problem?

Undocumented private enum declarations make the code harder to understand because readers have to infer intent from implementation details instead of reading a clear contract.

## How to fix it

Add a meaningful XML comment above the declaration. In most cases a short &lt;summary&gt; is the best fix.

## Examples

### Violation

```cs
/// <summary>
/// Hosts nested states
/// </summary>
internal class Container
{
    private enum Mode
    {
        Idle
    }
}
```

### Correction

```cs
/// <summary>
/// Hosts nested states.
/// </summary>
internal class Container
{
    /// <summary>
    /// Describes the current processing mode of the container
    /// </summary>
    private enum Mode
    {
        Idle
    }
}
```
