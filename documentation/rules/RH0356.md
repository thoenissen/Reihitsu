# RH0356 — No space after new for implicitly typed arrays

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH0356       |
| **Category** | Formatting   |
| **Severity** | Warning      |
| **Code Fix** | ✓            |

## Description

This rule ensures that no space after new for implicitly typed arrays.

## Why is this a problem?

Inconsistent formatting makes code harder to read and creates unnecessary noise in diffs.

## How to fix it

Adjust the affected formatting manually or apply the provided code fix.

## Examples

### Violation

```cs
internal class TestClass
{
    void Method()
    {
        int[] values = new [] { 1 };
    }
}
```

### Correction

```cs
internal class TestClass
{
    void Method()
    {
        int[] values = new[] { 1 };
    }
}
```
