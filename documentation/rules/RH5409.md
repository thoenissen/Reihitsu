# RH5409 — Final enum members must not have trailing commas

| Property | Value |
|----------|-------|
| **ID** | RH5409 |
| **Category** | Layout |
| **Severity** | Warning |
| **Code Fix** | ✓ |

## Description

This rule forbids a trailing comma after the final member in an enum declaration.

## Why is this a problem?

A trailing comma on the last enum member adds noise without improving readability. It also makes enum declarations inconsistent with the preferred formatting style enforced by this analyzer package.

Keeping the final member free of a trailing comma makes the end of the enum clearer and keeps formatting predictable.

## How to fix it

Remove the comma after the final enum member. If the member has an attached comment, keep the comment with the member and remove only the comma.

The code fix and formatter can apply this automatically.

## Examples

### Violation

```cs
internal enum Status
{
    Ready,
    Completed, // final value
}
```

### Correction

```cs
internal enum Status
{
    Ready,
    Completed // final value
}
```
