# RH0219 — Internal property names should be in PascalCase

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH0219       |
| **Category** | Naming       |
| **Severity** | Warning      |
| **Code Fix** | ✓            |

## Description

Checks that internal property names follow PascalCase naming conventions. Each word in the name should start with an uppercase letter, with no underscores or other separators.

## Why is this a problem?

PascalCase for properties is a fundamental C# naming convention that applies regardless of access modifier. Inconsistent naming reduces readability and makes the codebase harder to navigate.

## How to fix it

Rename the internal property to use PascalCase.

## Examples

### Violation

```cs
internal int count { get; set; }
```

### Correction

```cs
internal int Count { get; set; }
```
