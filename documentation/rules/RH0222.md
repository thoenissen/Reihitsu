# RH0222 — Tuple element names should be in camelCase

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH0222       |
| **Category** | Naming       |
| **Severity** | Warning      |
| **Code Fix** | ✗            |

## Description

Checks that tuple element names follow camelCase naming conventions. The first word should start with a lowercase letter, and each subsequent word should start with an uppercase letter.

## Why is this a problem?

camelCase for tuple element names is consistent with local variable and parameter naming conventions. Using PascalCase for tuple elements creates inconsistency and can be confused with property access.

## How to fix it

Rename the tuple element names to use camelCase.

## Examples

### Violation

```cs
(int Name, int Age) person;
```

### Correction

```cs
(int name, int age) person;
```
