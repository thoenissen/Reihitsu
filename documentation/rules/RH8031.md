# RH8031 — Extension declarations must have summary text

| Property | Value |
|----------|-------|
| **ID** | RH8031 |
| **Category** | Documentation |
| **Severity** | Warning |
| **Code Fix** | ❌ |

## Description

This rule requires each `extension(...)` or `extension<T>(...)` declaration to include a non-empty `<summary>` tag.

## Why is this a problem?

The extension declaration defines the receiver context for all members inside the block. Without summary text, callers and maintainers do not get clear intent for that extension scope.

## How to fix it

Add XML documentation to the extension declaration and ensure `<summary>` contains meaningful text.

## Examples

### Violation

```cs
public static class Extensions
{
    extension(string value)
    {
    }
}
```

### Correction

```cs
public static class Extensions
{
    /// <summary>Provides text-related extensions.</summary>
    extension(string value)
    {
    }
}
```
