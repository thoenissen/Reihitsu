# RH7310 — Empty regions should be removed

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH7310       |
| **Category** | Organization |
| **Severity** | Warning      |
| **Code Fix** | ✓            |

## Description

This rule checks whether a `#region` wraps any content. A region is reported only when the code between its `#region` and matching `#endregion` is truly empty — nothing but blank lines. Regions that wrap comments, disabled code (`#if false`), or other directives are treated as having content and are left alone, so the fix never silently deletes them.

## Why is this a problem?

An empty region adds visual noise and implies a structure that does not exist. Empty regions usually remain after their members are deleted or moved elsewhere. Removing them keeps the region layout honest, so every region that remains actually groups something. Comments inside a region are meaningful content — a region that documents intent with comments is preserved rather than removed.

## How to fix it

Delete the `#region` and `#endregion` directives for any region that no longer wraps any content. Nested empty regions are cleaned up one shell at a time: remove the innermost empty region first, then the outer region that only contained it.

## Examples

### Violation

```cs
public class Widget
{
    #region Methods

    #endregion // Methods
}
```

### Correction

```cs
public class Widget
{
}
```

### Not reported

A region whose body is only comments is treated as having content and is preserved:

```cs
public class Widget
{
    #region Properties

    // TODO: add properties later

    #endregion // Properties
}
```
