# RH7310 — Empty regions should be removed

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH7310       |
| **Category** | Organization |
| **Severity** | Warning      |
| **Code Fix** | ✓            |

## Description

This rule checks whether a `#region` groups any members. A region is reported when the code between its `#region` and matching `#endregion` contains no member declarations — only blank lines or comments.

## Why is this a problem?

An empty region adds visual noise and implies a structure that does not exist. Empty regions usually remain after their members are deleted or moved elsewhere. Removing them keeps the region layout honest, so every region that remains actually groups something.

## How to fix it

Delete the `#region` and `#endregion` directives for any region that no longer groups members. Nested empty regions are cleaned up one shell at a time: remove the innermost empty region first, then the outer region that only contained it.

## Examples

### Violation

```cs
public class Widget
{
    #region Methods

    #endregion // Methods

    #region Properties

    // TODO: add properties later

    #endregion // Properties
}
```

### Correction

```cs
public class Widget
{
}
```
