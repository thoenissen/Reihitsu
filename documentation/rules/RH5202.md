# RH5202 — Raw string literals should be formatted correctly

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH5202       |
| **Category** | Layout |
| **Severity** | Warning      |
| **Code Fix** | ✓            |

## Description

This rule ensures that multi-line raw string literals have their opening and closing quote markers (`"""`) aligned to the same column. The content lines are adjusted uniformly to maintain consistent indentation.

## Why is this a problem?

When the closing marker of a raw string literal is not vertically aligned with the opening marker, the code becomes harder to read and the effective content of the string may not match the developer's intent, since the closing marker column determines the common whitespace prefix that is stripped from all content lines.

## How to fix it

Align the closing quote markers (`"""`) to the same column as the opening markers. Adjust all content lines by the same indentation delta. The code fix will automatically reformat the raw string.

## Examples

### Violation

```cs
var a = """
    Test
    """;
```

### Correction

```cs
var a = """
        Test
        """;
```

### Interpolated violation

```cs
var a = $"""
    Hello {name}
    """;
```

### Interpolated correction

```cs
var a = $"""
         Hello {name}
         """;
```

### UTF-8 violation

```cs
var a = """
    Test
    """u8;
```

### UTF-8 correction

```cs
var a = """
        Test
        """u8;
```

> Multi-line UTF-8 raw string literals (the `u8` suffix) are aligned the same way as regular multi-line raw string literals.
