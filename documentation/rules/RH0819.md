# RH0819 — Empty record structs should use semicolon declarations

| Property | Value |
|----------|-------|
| **ID** | RH0819 |
| **Category** | Formatting |
| **Severity** | Warning |
| **Code Fix** | ✓ |

## Description

This rule prefers semicolon declarations for empty record structs.

Brace-based empty record struct bodies add noise when the declaration contains no members and the language version supports semicolon syntax.

## Why is this a problem?

An empty record struct already expresses its full intent on the declaration line. The empty body adds visual overhead without adding behavior.

Using a semicolon declaration keeps declaration-only record structs brief and consistent with the preferred formatting style.

## How to fix it

Replace the empty brace body with a trailing semicolon.

The code fix and formatter only apply this when the body is truly empty. Comments, directives, and other meaningful trivia inside the braces are preserved by leaving the declaration unchanged.

## Examples

### Violation

```cs
internal record struct Example
{
}
```

### Correction

```cs
internal record struct Example;
```
