# RH2105 — Nested interfaces should not be used

| Property | Value |
|----------|-------|
| **ID** | RH2105 |
| **Category** | Design |
| **Severity** | Warning |
| **Code Fix** | ❌ |

## Description

Do not declare an interface inside another type.

## Why is this a problem?

Nested interfaces are easy to miss and increase coupling to the containing type. Top-level interfaces are easier to discover, review, and implement.

## How to fix it

Move the nested interface to a top-level declaration.

## Examples

### Violation

```cs
namespace Example;

internal class Container
{
    private interface IProcessor
    {
        void Run();
    }
}
```

### Correction

```cs
namespace Example;

internal class Container
{
}

internal interface IProcessor
{
    void Run();
}
```
