# RH0408 - Non-private records must be documented

| Property | Value |
|----------|-------|
| **ID** | RH0408 |
| **Category** | Documentation |
| **Severity** | Warning |
| **Code Fix** | ❌ |

## Description

This rule requires non-private record declarations to have XML documentation. A declaration counts as documented when it has a &lt;summary&gt; tag or uses &lt;inheritdoc/&gt;.

## Why is this a problem?

Undocumented non-private record declarations make the code harder to understand because readers have to infer intent from implementation details instead of reading a clear contract.

## How to fix it

Add a meaningful XML comment above the declaration. In most cases a short &lt;summary&gt; is the best fix.

## Examples

### Violation

```cs
internal record Customer(string Name);
```

### Correction

```cs
/// <summary>
/// Represents a customer in the billing workflow
/// </summary>
internal record Customer(string Name);
```
