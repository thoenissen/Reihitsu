# RH0114 — Classes should not use parameterized primary constructors

| Property | Value |
|----------|-------|
| **ID** | RH0114 |
| **Category** | Design |
| **Severity** | Warning |
| **Code Fix** | ❌ |

## Description

Disallows primary constructors with parameters on class declarations.

## Why is this a problem?

Parameterized primary constructors on classes can hide initialization flow in the type header and make construction patterns less consistent across a codebase. Using explicit constructors keeps initialization logic easier to read and maintain.

## How to fix it

Remove constructor parameters from the class declaration and move them to an explicit constructor in the class body.

## Examples

### Violation

```cs
namespace Example;

internal class Service(int timeout)
{
}
```

### Correction

```cs
namespace Example;

internal class Service
{
    internal Service(int timeout)
    {
    }
}
```
