# RH0370 — Element documentation header must be preceded by blank line

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH0370       |
| **Category** | Formatting   |
| **Severity** | Warning      |
| **Code Fix** | ✓            |

## Description

This rule ensures that element documentation header must be preceded by blank line.

## Why is this a problem?

Inconsistent formatting makes code harder to read and creates unnecessary noise in diffs.

## How to fix it

Adjust the affected formatting manually or apply the provided code fix.

## Examples

### Violation

```cs
internal class TestClass
{
    void First()
    {
    }
    /// <summary>
    /// Summary.
    /// </summary>
    void Second()
    {
    }
}
```

### Correction

```cs
internal class TestClass
{
    void First()
    {
    }

    /// <summary>
    /// Summary.
    /// </summary>
    void Second()
    {
    }
}
```
