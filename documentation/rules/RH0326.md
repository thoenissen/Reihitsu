# RH0326 — Expression style constructors should not be used

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH0326       |
| **Category** | Formatting   |
| **Severity** | Warning      |
| **Code Fix** | ✗            |

## Description

This rule disallows expression-bodied constructors (constructors using `=>`). All constructors should use block bodies for consistency and clarity.

## Why is this a problem?

Expression-bodied constructors can obscure initialization logic and create inconsistency when some constructors use block bodies and others use expression bodies. Requiring block bodies ensures a uniform coding style and makes constructor logic explicit.

## How to fix it

Replace the expression-bodied constructor with a block body containing the assignment or initialization statements.

## Examples

### Violation

```cs
public MyClass(int x) => _x = x;
```

### Correction

```cs
public MyClass(int x)
{
    _x = x;
}
```
