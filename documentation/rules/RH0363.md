# RH0363 — Opening brace must not be followed by blank line

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH0363       |
| **Category** | Formatting   |
| **Severity** | Warning      |
| **Code Fix** | ✓            |

## Description

The first line of content inside a block must not be separated from the opening brace by a blank line.

## Why is this a problem?

A blank line immediately after an opening brace adds vertical whitespace with no semantic value, pushing the actual content further from its context.

## How to fix it

Remove the blank line that follows the opening brace. The code fix applies this change automatically.

## Examples

### Violation

```cs
internal class TestClass
{
    void Method()
    {

        int value = 0;
    }
}
```

### Correction

```cs
internal class TestClass
{
    void Method()
    {
        int value = 0;
    }
}
```
