# RH7309 — Regions should follow a fixed category order

| Property | Value |
|----------|-------|
| **ID** | RH7309 |
| **Category** | Organization |
| **Severity** | Warning |
| **Code Fix** | ✓ |

## Description

This rule enforces a fixed order for the top-level region categories inside a type. Regions must appear grouped by category in the following order:

1. **Custom regions** — regions with a self-defined name that describe the type's own surface, such as `Constants`, `Fields`, `Constructor`, `Properties` or `Methods`.
2. **Base-type override regions** — regions named after a base type that group the members overriding that base type.
3. **Interface implementation regions** — regions named after an implemented interface that group the members implementing it.

A region's category is derived from its description: a description that matches an implemented interface is an interface region, a description that matches a base type is a base-type override region, and everything else is a custom region.

The rule reports any region whose category appears before a region of an earlier category — for example, an interface region placed before a custom region. Ordering *within* a category is out of scope and stays with the category-specific rules (RH7401–RH7404 for base-type override regions and RH7409–RH7412 for interface regions).

## Why is this a problem?

A predictable region layout makes a type scannable: readers see the type's own surface first, then what it overrides from its base type, and finally what it fulfills from its interfaces. When the categories are interleaved or reversed, that reading order breaks down and the value of grouping members into regions is lost.

## How to fix it

Move whole `#region … #endregion` blocks so that all custom regions come first, followed by the base-type override regions, followed by the interface implementation regions. The code fix performs this reordering automatically.

## Examples

### Violation

```cs
public class Widget : WidgetBase, IWidget
{
    #region IWidget

    public void Render()
    {
    }

    #endregion // IWidget

    #region Methods

    public void Refresh()
    {
    }

    #endregion // Methods
}
```

### Correction

```cs
public class Widget : WidgetBase, IWidget
{
    #region Methods

    public void Refresh()
    {
    }

    #endregion // Methods

    #region IWidget

    public void Render()
    {
    }

    #endregion // IWidget
}
```
