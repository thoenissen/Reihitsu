# RH0443 - Generic type parameter documentation must have text

| Property | Value |
|----------|-------|
| **ID** | RH0443 |
| **Category** | Documentation |
| **Severity** | Warning |
| **Code Fix** | ❌ |

## Description

This rule requires each `<typeparam>` tag to contain descriptive text.

## Why is this a problem?

An empty `<typeparam>` tag suggests the generic contract is documented when it is not. That weakens IntelliSense and generated documentation.

## How to fix it

Add a short description for every generic type parameter.

## Examples

### Violation

```cs
/// <summary>
/// Represents a repository
/// </summary>
/// <typeparam name="T"></typeparam>
internal class Repository<T>
{
}
```

### Correction

```cs
/// <summary>
/// Represents a repository
/// </summary>
/// <typeparam name="T">The entity type stored by the repository</typeparam>
internal class Repository<T>
{
}
```
