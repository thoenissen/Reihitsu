# RH0327 — Expression style get-only properties should be single lined

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH0327       |
| **Category** | Formatting   |
| **Severity** | Warning      |
| **Code Fix** | ✔️            |

## Description

This rule checks that expression-bodied get-only properties are written on a single line. If you choose to use an expression-bodied property, the entire declaration including the `=>` and expression should not span multiple lines.

## Why is this a problem?

Splitting an expression-bodied property across multiple lines defeats the purpose of using the concise expression syntax. A multi-line expression-bodied property is harder to read than either a single-line version or a full property with a `get` accessor block.

## How to fix it

Place the entire expression-bodied property on a single line, or convert it to a full property with a `get` accessor block if the expression is too long.

## Examples

### Violation

```cs
public int Value
    => 42;
```

### Correction

```cs
public int Value => 42;
```
