# RH0302 — Object initializer should be formatted correctly

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH0302       |
| **Category** | Formatting   |
| **Severity** | Warning      |
| **Code Fix** | ✓            |

## Description

This rule ensures that object initializer blocks are formatted with proper alignment. The opening brace, each property assignment, and the closing brace should be placed on separate lines with correct indentation.

## Why is this a problem?

Inline or poorly formatted object initializers reduce readability, especially when multiple properties are being set. Consistent formatting makes the structure of the initialization immediately clear and easier to review.

## How to fix it

Place the opening brace on a new line aligned with the `new` keyword, each property assignment on its own line with proper indentation, and the closing brace on a new line. The code fix will reformat the initializer automatically.

## Examples

### Violation

```cs
var obj = new Foo { Name = "test",
    Age = 5 };
```

### Correction

```cs
var obj = new Foo
          {
              Name = "test",
              Age = 5
          };
```
