# RH0118 — Nested records should not be used

| Property | Value |
|----------|-------|
| **ID** | RH0118 |
| **Category** | Design |
| **Severity** | Warning |
| **Code Fix** | ❌ |

## Description

Do not declare a record or record struct inside another type.

## Why is this a problem?

Nested record declarations reduce discoverability and make reuse less obvious. Keeping records top-level improves clarity and keeps ownership boundaries explicit.

## How to fix it

Move the nested record declaration to a top-level declaration.

## Examples

### Violation

```cs
namespace Example;

internal class Container
{
    private record Details(int Value);
}
```

### Correction

```cs
namespace Example;

internal class Container
{
}

internal record Details(int Value);
```
