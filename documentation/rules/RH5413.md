# RH5413 — Empty structs should use semicolon declarations

| Property | Value |
|----------|-------|
| **ID** | RH5413 |
| **Category** | Layout |
| **Severity** | Warning |
| **Code Fix** | ✓ |

## Description

This rule prefers semicolon declarations for empty structs.

Brace-based empty struct bodies add noise when the declaration contains no members and the language version supports semicolon syntax.

## Why is this a problem?

An empty struct body adds visual weight without contributing implementation detail.

Using a semicolon declaration keeps simple marker structs and placeholder value types compact and easier to scan.

## How to fix it

Replace the empty brace body with a trailing semicolon.

The code fix and formatter only apply this when the body is truly empty. Comments, directives, and other meaningful trivia inside the braces are preserved by leaving the declaration unchanged.

## Examples

### Violation

```cs
internal struct Example
{
}
```

### Correction

```cs
internal struct Example;
```
