# RH0613 — Using static directives must be placed at correct position

| Property | Value |
|----------|-------|
| **ID** | RH0613 |
| **Category** | Ordering |
| **Severity** | Warning |
| **Code Fix** | ✓ |

## Description

This rule requires `using static` directives to appear after regular `using` directives and before alias directives.

## Why is this a problem?

Static imports are easier to notice when they are grouped consistently. Mixing them into regular imports or alias blocks makes the using section harder to scan.

## How to fix it

Move `using static` directives below regular imports and above alias imports.

## Examples

### Violation

```cs
using static System.Math;
using MyProject.Common;
```

### Correction

```cs
using MyProject.Common;
using static System.Math;
```
