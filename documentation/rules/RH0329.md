# RH0329 — Logical expressions should be formatted correctly

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH0329       |
| **Category** | Formatting   |
| **Severity** | Warning      |
| **Code Fix** | ✓            |

## Description

This rule ensures that multi-line logical expressions place the logical operator (`&&`, `||`) at the beginning of the continuation line rather than at the end of the preceding line. This makes the logical structure of compound conditions easier to follow.

## Why is this a problem?

When logical operators are placed at the end of a line, it is easy to miss them, making it harder to understand the overall condition. Leading operators on continuation lines make the logical structure immediately visible when scanning the left edge of the code.

## How to fix it

Move the logical operator from the end of the line to the beginning of the next line. The code fix will automatically reformat the expression.

## Examples

### Violation

```cs
if (condition1 &&
    condition2 ||
    condition3)
{
}
```

### Correction

```cs
if (condition1
    && condition2
    || condition3)
{
}
```
