# RH5031 — Region directives should be preceded by a blank line

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH5031       |
| **Category** | Layout |
| **Severity** | Warning      |
| **Code Fix** | ✓            |

## Description

A `#region` or `#endregion` directive should have a blank line before it, so the directive is visually separated from the preceding code.

## Why is this a problem?

Without a blank line in front of it, a region marker merges with the preceding member and the grouping is harder to see at a glance.

## How to fix it

Add a blank line before the directive. The only exception is when the directive directly follows an opening brace: no blank line is required between the `{` and the directive, keeping the rule consistent with RH5022. The code fix inserts the missing blank line automatically.

## Examples

### Violation

```cs
internal class TestClass
{
    private int _a;
    #region Helpers

    private int _b;
}
```

### Correction

```cs
internal class TestClass
{
    private int _a;

    #region Helpers

    private int _b;
}
```
