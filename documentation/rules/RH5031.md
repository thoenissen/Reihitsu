# RH5031 — Region directives must be surrounded by blank lines

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH5031       |
| **Category** | Layout |
| **Severity** | Warning      |
| **Code Fix** | ✓            |

## Description

A `#region` directive must have a blank line before and after it, so the directive is visually separated from the surrounding members.

## Why is this a problem?

Without the surrounding blank lines, the `#region` marker merges with the adjacent member and the grouping is harder to see at a glance.

## How to fix it

Add a blank line before and after the `#region` directive. The only exception is when the directive directly follows an opening brace: no blank line is added between the `{` and the `#region`, keeping the rule consistent with the brace blank-line rules. The code fix inserts the missing blank lines automatically.

## Examples

### Violation

```cs
internal class TestClass
{
    private int _a;
    #region Helpers
    private int _b;
}
```

### Correction

```cs
internal class TestClass
{
    private int _a;

    #region Helpers

    private int _b;
}
```
