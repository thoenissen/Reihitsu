# RH0121 — Razor @code blocks should not be used

| Property | Value |
|----------|-------|
| **ID** | RH0121 |
| **Category** | Design |
| **Severity** | Warning |
| **Code Fix** | ❌ |

## Description

Do not use `@code { ... }` blocks in Razor component files.

## Why is this a problem?

Keeping component logic inside `.razor` markup mixes rendering and behavior in the same file. Moving logic to `.razor.cs` code-behind partial classes improves separation of concerns and maintainability.

## How to fix it

Move logic from the `@code` block into a matching `.razor.cs` partial class for the component.

## Examples

### Violation

```razor
@page "/counter"

<h1>@CurrentCount</h1>

@code {
    private int CurrentCount;
}
```

### Correction

```razor
@page "/counter"

<h1>@CurrentCount</h1>
```

```cs
public partial class Counter
{
    private int CurrentCount;
}
```
