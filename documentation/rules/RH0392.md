# RH0392 — Statement lambda opening braces should be aligned

| Property | Value |
|----------|-------|
| **ID** | RH0392 |
| **Category** | Formatting |
| **Severity** | Warning |
| **Code Fix** | ✓ |

## Description

This rule requires statement lambdas with block bodies to align the opening brace under the lambda argument anchor.

## Why is this a problem?

When the lambda body starts farther to the right than the lambda arguments, the shape of the statement becomes harder to scan. Keeping the opening brace aligned with the lambda anchor makes multiline lambdas easier to read consistently.

## How to fix it

Move the opening brace of the lambda block so it starts under the lambda anchor. For simple lambdas, use the parameter name as the anchor. For parenthesized lambdas, use the opening parenthesis. For `async` lambdas, use the `async` keyword.

## Examples

### Violation

```cs
Func<object, object> projector = obj =>
                                          {
                                              return obj;
                                          };

Func<object, int, Task<object>> asyncProjector = async (obj, index) =>
                                                                            {
                                                                                return await Task.FromResult(obj);
                                                                            };
```

### Correction

```cs
Func<object, object> projector = obj =>
                                 {
                                     return obj;
                                 };

Func<object, int, Task<object>> asyncProjector = async (obj, index) =>
                                                 {
                                                     return await Task.FromResult(obj);
                                                 };
```
