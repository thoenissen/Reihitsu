# RH0702 — Files must start with configured copyright header

| Property | Value |
|----------|-------|
| **ID** | RH0702 |
| **Category** | Analyzer |
| **Severity** | Warning |
| **Code Fix** | ✔ |

## Description

When the `Copyright` section is configured in `reihitsu.json`, this rule requires each analyzed source file to start with the configured header text.

## Why is this a problem?

Missing or inconsistent file headers can lead to unclear ownership or licensing information and make repository-wide legal requirements hard to enforce.

## How to fix it

Define a `Copyright.copyrightText` template in `reihitsu.json` and ensure each file starts with the resolved text. You can use `{fileName}` and additional placeholders that are configured in the same section.

## Examples

### Configuration

```cs
// reihitsu.json
{
  "Copyright": {
    "copyrightText": "// Copyright (c) {companyName}.\n// Licensed under {licenseName}. See {licenseFile}.",
    "companyName": "Example Software",
    "licenseName": "MIT",
    "licenseFile": "LICENSE"
  }
}
```

### Violation

```cs
namespace Example.Orders;

internal sealed class InvoicePrinter
{
}
```

### Correction

```cs
// Copyright (c) Example Software.
// Licensed under MIT. See LICENSE.
namespace Example.Orders;

internal sealed class InvoicePrinter
{
}
```
