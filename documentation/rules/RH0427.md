# RH0427 - Non-private fields must be documented

| Property | Value |
|----------|-------|
| **ID** | RH0427 |
| **Category** | Documentation |
| **Severity** | Warning |
| **Code Fix** | ❌ |

## Description

This rule requires non-private field declarations to have XML documentation. A declaration counts as documented when it has a &lt;summary&gt; tag or uses &lt;inheritdoc/&gt;.

## Why is this a problem?

Undocumented non-private field declarations make the code harder to understand because readers have to infer intent from implementation details instead of reading a clear contract.

## How to fix it

Add a meaningful XML comment above the declaration. In most cases a short &lt;summary&gt; is the best fix.

## Examples

### Violation

```cs
/// <summary>
/// Stores counters
/// </summary>
internal class CounterStore
{
    internal int currentCount;
}
```

### Correction

```cs
/// <summary>
/// Stores counters
/// </summary>
internal class CounterStore
{
    /// <summary>
    /// Stores the currently processed item count
    /// </summary>
    internal int currentCount;
}
```
