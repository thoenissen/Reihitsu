# RH0373 — Braces must not be omitted from multi-line child statements

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH0373       |
| **Category** | Formatting   |
| **Severity** | Warning      |
| **Code Fix** | ✓            |

## Description

A child statement that spans multiple lines must always be enclosed in braces, even when only one logical statement is present.

## Why is this a problem?

A multi-line brace-less body is visually indistinguishable from a block body and is easy to misread, especially when the body contains a method call with multiple arguments split across lines.

## How to fix it

Wrap the multi-line child statement in braces. The code fix applies this change automatically.

## Examples

### Violation

```cs
internal class TestClass
{
    void Method()
    {
        if (true)
            Other(1);
    }
}
```

### Correction

```cs
internal class TestClass
{
    void Method()
    {
        if (true)
        {
            Other(1);
        }
    }
}
```
