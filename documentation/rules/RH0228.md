# RH0228 — Static readonly field names should be in PascalCase

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH0228       |
| **Category** | Naming       |
| **Severity** | Warning      |
| **Code Fix** | ✓            |

## Description

Checks that `static readonly` field names follow PascalCase naming conventions regardless of their access modifier. This keeps field-like constants aligned with common .NET naming conventions and avoids conflicting with RH0212 for private instance fields.

## Why is this a problem?

`static readonly` fields are effectively shared values and are typically treated like named constants in C# codebases. Using `_camelCase` or other naming styles makes them harder to distinguish from ordinary mutable fields.

## How to fix it

Rename the `static readonly` field to use PascalCase.

## Examples

### Violation

```cs
private static readonly int _maxValue = 100;
internal static readonly string value = string.Empty;
```

### Correction

```cs
private static readonly int MaxValue = 100;
internal static readonly string Value = string.Empty;
```
