# RH0401 — The inheritdoc tag should be used if possible

| Property     | Value         |
|--------------|---------------|
| **ID**       | RH0401        |
| **Category** | Documentation |
| **Severity** | Warning       |
| **Code Fix** | ✓             |

## Description

The `<inheritdoc/>` tag should be used when overriding or implementing members that already have XML documentation on the base type or interface. This rule flags members that duplicate documentation instead of inheriting it.

## Why is this a problem?

Repeating XML documentation on overridden or implemented members leads to duplication and increases the maintenance burden. When the base documentation changes, all copies must be updated manually, which is error-prone and often results in inconsistencies.

## How to fix it

Replace the full XML documentation comment with a single `<inheritdoc/>` tag.

## Examples

### Violation

```cs
/// <summary>
/// Returns a string representation.
/// </summary>
public override string ToString()
{
    return "";
}
```

### Correction

```cs
/// <inheritdoc/>
public override string ToString()
{
    return "";
}
```
