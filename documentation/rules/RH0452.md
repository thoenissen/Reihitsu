# RH0452 — Files must start with configured XML-style copyright header

| Property | Value |
|----------|-------|
| **ID** | RH0452 |
| **Category** | Documentation |
| **Severity** | Warning |
| **Code Fix** | ✔ |

## Description

When the `Copyright` section is configured in `reihitsu.json`, this rule requires each analyzed source file to start with the configured XML-style `//` copyright header text.

## Why is this a problem?

Missing or inconsistent file headers can lead to unclear ownership or licensing information and make repository-wide legal requirements hard to enforce.

## How to fix it

Define a `Copyright.copyrightText` template in `reihitsu.json` and ensure each file starts with the resolved XML-style `//` copyright text. The template must use `// ` line comments, begin with `<copyright ...>`, and end with `</copyright>`.

## Examples

### Configuration

```json
// reihitsu.json
{
  "Copyright": {
    "copyrightText": "// <copyright file=\"{fileName}\" company=\"{companyName}\">\n// Copyright (c) {companyName}. All rights reserved.\n// Licensed under the {licenseName} license. See {licenseFile}.\n// </copyright>",
    "companyName": "Example Software",
    "licenseName": "MIT",
    "licenseFile": "LICENSE"
  }
}
```

### Violation

```cs
namespace Example.Orders;

internal sealed class InvoicePrinter
{
}
```

### Correction

```cs
// <copyright file="InvoicePrinter.cs" company="Example Software">
// Copyright (c) Example Software. All rights reserved.
// Licensed under the MIT license. See LICENSE.
// </copyright>
namespace Example.Orders;

internal sealed class InvoicePrinter
{
}
```
