# RH0403 - Private classes must be documented

| Property | Value |
|----------|-------|
| **ID** | RH0403 |
| **Category** | Documentation |
| **Severity** | Warning |
| **Code Fix** | ❌ |

## Description

This rule requires private class declarations to have XML documentation. A declaration counts as documented when it has a &lt;summary&gt; tag or uses &lt;inheritdoc/&gt;.

## Why is this a problem?

Undocumented private class declarations make the code harder to understand because readers have to infer intent from implementation details instead of reading a clear contract.

## How to fix it

Add a meaningful XML comment above the declaration. In most cases a short &lt;summary&gt; is the best fix.

## Examples

### Violation

```cs
/// <summary>
/// Hosts nested helpers.
/// </summary>
internal class Container
{
    private class Worker
    {
    }
}
```

### Correction

```cs
/// <summary>
/// Hosts nested helpers
/// </summary>
internal class Container
{
    /// <summary>
    /// Performs the background work for the container
    /// </summary>
    private class Worker
    {
    }
}
```
