# RH0310 — return-Statement should be preceded by a blank line

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH0310       |
| **Category** | Formatting   |
| **Severity** | Warning      |
| **Code Fix** | ✔️            |

## Description

This rule checks that `return` statements are preceded by a blank line. The rule does not apply when the `return` statement is the first statement in a block.

## Why is this a problem?

Separating statements with blank lines improves code readability by visually grouping related logic. A blank line before a `return` statement makes the exit point of a method clearly visible.

## How to fix it

Add a blank line before the `return` statement. The rule does not apply when the statement is the first statement in a block.

## Examples

### Violation

```cs
var result = Calculate();
return result;
```

### Correction

```cs
var result = Calculate();

return result;
```
