# RH0354 — Dereference and access-of symbols must be spaced correctly

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH0354       |
| **Category** | Formatting   |
| **Severity** | Warning      |
| **Code Fix** | ✓            |

## Description

This rule ensures that dereference and access-of symbols must be spaced correctly.

## Why is this a problem?

Inconsistent formatting makes code harder to read and creates unnecessary noise in diffs.

## How to fix it

Adjust the affected formatting manually or apply the provided code fix.

## Examples

### Violation

```cs
internal class TestClass
{
    unsafe void Method()
    {
        int value = 0;
        int* pointer = & value;
    }
}
```

### Correction

```cs
internal class TestClass
{
    unsafe void Method()
    {
        int value = 0;
        int* pointer = &value;
    }
}
```
