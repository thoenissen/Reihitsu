# RH0354 — Dereference and access-of symbols must be spaced correctly

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH0354       |
| **Category** | Formatting   |
| **Severity** | Warning      |
| **Code Fix** | ✓            |

## Description

The address-of `&` and dereference `*` pointer operators must not be followed by a space.

## Why is this a problem?

A space after a pointer operator visually separates it from the operand, making unsafe pointer expressions harder to read.

## How to fix it

Remove the space immediately after the `&` or `*` operator. The code fix applies this change automatically.

## Examples

### Violation

```cs
internal class TestClass
{
    unsafe void Method()
    {
        int value = 0;
        int* pointer = & value;
    }
}
```

### Correction

```cs
internal class TestClass
{
    unsafe void Method()
    {
        int value = 0;
        int* pointer = &value;
    }
}
```
