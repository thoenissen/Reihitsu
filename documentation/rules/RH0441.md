# RH0441 - Generic type parameter documentation must match type parameters

| Property | Value |
|----------|-------|
| **ID** | RH0441 |
| **Category** | Documentation |
| **Severity** | Warning |
| **Code Fix** | ❌ |

## Description

This rule requires `<typeparam>` tags to match the declaration's generic type parameters and appear in the same order.

## Why is this a problem?

If generic parameter documentation is out of order or refers to the wrong parameter, the XML comments no longer describe the declaration correctly.

## How to fix it

Rename or reorder the `<typeparam>` tags so they line up exactly with the generic type parameter list.

## Examples

### Violation

```cs
/// <summary>
/// Represents a pair
/// </summary>
/// <typeparam name="TSecond">The second value</typeparam>
/// <typeparam name="TFirst">The first value</typeparam>
internal class Pair<TFirst, TSecond>
{
}
```

### Correction

```cs
/// <summary>
/// Represents a pair
/// </summary>
/// <typeparam name="TFirst">The first value</typeparam>
/// <typeparam name="TSecond">The second value</typeparam>
internal class Pair<TFirst, TSecond>
{
}
```
