# RH0609 — Using alias directives must be ordered alphabetically by alias name

| Property | Value |
|----------|-------|
| **ID** | RH0609 |
| **Category** | Ordering |
| **Severity** | Warning |
| **Code Fix** | ✓ |

## Description

This rule requires using-alias directives to be sorted alphabetically by alias name.

## Why is this a problem?

When aliases are unordered, it takes longer to find the one you need. Alphabetical ordering keeps alias blocks predictable.

## How to fix it

Sort `using Alias = ...;` directives by the alias identifier.

## Examples

### Violation

```cs
using StringList = System.Collections.Generic.List<string>;
using IntList = System.Collections.Generic.List<int>;
```

### Correction

```cs
using IntList = System.Collections.Generic.List<int>;
using StringList = System.Collections.Generic.List<string>;
```
