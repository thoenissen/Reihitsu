# RH0108 — Debug.Assert must provide message text

| Property | Value |
|----------|-------|
| **ID** | RH0108 |
| **Category** | Design |
| **Severity** | Warning |
| **Code Fix** | ❌ |

## Description

Calls to `Debug.Assert` must include message text in addition to the asserted condition.

## Why is this a problem?

An assertion failure without a message gives too little context to understand what went wrong.

## How to fix it

Pass a descriptive message as the second argument to `Debug.Assert`.
