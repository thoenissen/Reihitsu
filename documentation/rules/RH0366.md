# RH0366 — Closing brace must not be preceded by blank line

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH0366       |
| **Category** | Formatting   |
| **Severity** | Warning      |
| **Code Fix** | ✓            |

## Description

This rule ensures that closing brace must not be preceded by blank line.

## Why is this a problem?

Inconsistent formatting makes code harder to read and creates unnecessary noise in diffs.

## How to fix it

Adjust the affected formatting manually or apply the provided code fix.

## Examples

### Violation

```cs
internal class TestClass
{
    void Method()
    {
        int value = 0;

    }
}
```

### Correction

```cs
internal class TestClass
{
    void Method()
    {
        int value = 0;
    }
}
```
