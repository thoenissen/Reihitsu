# RH0221 — Local variable names should be in camelCase

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH0221       |
| **Category** | Naming       |
| **Severity** | Warning      |
| **Code Fix** | ✓            |

## Description

Checks that local variable names follow camelCase naming conventions. The first word should start with a lowercase letter, and each subsequent word should start with an uppercase letter.

## Why is this a problem?

camelCase for local variables is a fundamental C# naming convention. Using PascalCase or other conventions for locals makes it harder to distinguish them from types, properties, and methods.

## How to fix it

Rename the local variable to use camelCase.

## Examples

### Violation

```cs
var MyValue = 42;
```

### Correction

```cs
var myValue = 42;
```
