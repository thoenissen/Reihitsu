# RH5417 — Member declaration braces must not share a line

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH5417       |
| **Category** | Layout |
| **Severity** | Warning      |
| **Code Fix** | ✓            |

## Description

The body braces of a member declaration must not share a line. This applies to methods, constructors, destructors, operators, conversion operators, and property, event, and indexer accessors. Both the empty form (`{ }`) and the populated single-line form (`{ return 1; }`) are reported.

## Why is this a problem?

Reihitsu uses Allman-style braces, where the opening and closing brace of a body each sit on their own line. Collapsing a member body onto a single line breaks that consistency, hides the body's structure, and makes it harder to add statements or read the member at a glance.

## How to fix it

Place the opening brace and the closing brace on their own lines. The code fix applies this change automatically.

Expression-bodied members (`=> ...`) and auto-property accessor lists (`{ get; set; }`) are not affected by this rule.

## Examples

### Violation

```cs
public class C
{
    protected virtual void Write() { }

    public int GetValue() { return 1; }
}
```

### Correction

```cs
public class C
{
    protected virtual void Write()
    {
    }

    public int GetValue()
    {
        return 1;
    }
}
```
