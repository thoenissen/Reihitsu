# RH0001 — Not operator should not be used

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH0001       |
| **Category** | Clarity      |
| **Severity** | Warning      |
| **Code Fix** | ✓            |

## Description

The logical negation operator `!` should not be used for boolean negation. This rule enforces using `== false` instead of the `!` operator to improve readability.

## Why is this a problem?

The `!` operator is easy to overlook, especially in complex conditional expressions with multiple operands. Using `== false` makes the negation explicit and immediately visible, reducing the risk of misreading the intent of the code.

## How to fix it

Replace `!condition` with `condition == false`.

## Examples

### Violation

```cs
if (!isValid)
{
}
```

### Correction

```cs
if (isValid == false)
{
}
```
