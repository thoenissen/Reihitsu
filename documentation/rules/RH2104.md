# RH2104 — Nested structs should not be used

| Property | Value |
|----------|-------|
| **ID** | RH2104 |
| **Category** | Design |
| **Severity** | Warning |
| **Code Fix** | ❌ |

## Description

Do not declare a struct inside another type.

## Why is this a problem?

Nested structs are harder to discover and reuse. They also hide declarations inside unrelated types and make ownership less obvious during maintenance.

## How to fix it

Move the nested struct to a top-level declaration.

## Examples

### Violation

```cs
namespace Example;

internal class Container
{
    private struct Details
    {
        internal int Value;
    }
}
```

### Correction

```cs
namespace Example;

internal class Container
{
}

internal struct Details
{
    internal int Value;
}
```
