# RH0384 — Generic type constraints should be on their own line with proper indentation

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH0384       |
| **Category** | Formatting   |
| **Severity** | Warning      |
| **Code Fix** | ✓            |

## Description

This rule ensures that generic type constraints are written on their own line and indented one level from the declaration.

## Why is this a problem?

Constraint clauses become harder to scan when they remain on the declaration line or are not aligned consistently. Putting each `where` clause on its own indented line keeps generic declarations easier to read and maintain.

## How to fix it

Move each `where` clause to its own line directly below the declaration and indent it one level further than the declaration line. You can do this manually or apply the provided code fix.

## Examples

### Violation

```cs
internal class Example<T> where T : class
{
}

internal class Container
{
    void Method<TKey, TValue>()
    where TKey : notnull
    where TValue : class
    {
    }
}
```

### Correction

```cs
internal class Example<T>
    where T : class
{
}

internal class Container
{
    void Method<TKey, TValue>()
        where TKey : notnull
        where TValue : class
    {
    }
}
```
