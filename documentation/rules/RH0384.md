# RH0384 — C# source files should be encoded as UTF-8 with BOM

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH0384       |
| **Category** | Formatting   |
| **Severity** | Warning      |
| **Code Fix** | ✓            |

## Description

This rule requires C# source files to be encoded as UTF-8 with a byte order mark (BOM).

## Why is this a problem?

Using one encoding consistently avoids editor, diff, and tooling inconsistencies when source files move between environments.

## How to fix it

Re-save the file as UTF-8 with BOM. The code fix updates the document encoding without changing the source text.

## Examples

### Violation

```cs
// File saved as UTF-8 without BOM
internal class TestClass
{
}
```

### Correction

```cs
// File saved as UTF-8 with BOM
internal class TestClass
{
}
```
