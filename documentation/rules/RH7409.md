# RH7409 — Interface implementation methods should be grouped by interface regions

| Property | Value |
|----------|-------|
| **ID** | RH7409 |
| **Category** | Organization |
| **Severity** | Warning |
| **Code Fix** | ❌ |

## Description

This rule requires each method that implements an interface method to appear inside a top-level `#region` whose label matches the interface that declares the member.

## Why is this a problem?

Members implementing an interface are easier to scan when they are grouped by the interface they belong to. Generic region names such as `Methods` hide that relationship and make types that implement several interfaces harder to navigate.

## How to fix it

Move each interface implementation method into a top-level region named after the implemented interface. Use the interface name only, such as `IDisposable`. This applies to both explicit and implicit implementations.

## Examples

### Violation

```cs
internal class TestClass : IDisposable
{
    #region Methods

    public void Dispose()
    {
    }

    #endregion // Methods
}
```

### Correction

```cs
internal class TestClass : IDisposable
{
    #region IDisposable

    public void Dispose()
    {
    }

    #endregion // IDisposable
}
```
