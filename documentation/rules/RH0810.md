# RH0810 — Simple auto-properties should be single lined

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH0810       |
| **Category** | Formatting   |
| **Severity** | Warning      |
| **Code Fix** | ✓ |

## Description

This rule checks that simple auto-properties — brace-based property declarations whose accessors have no body — are written on a single line. A property is considered eligible when it has no attributes and the full declaration can stay readable on one line after formatting.

Expression-bodied properties (`=>`) are out of scope for this rule; they are covered by [RH0327](RH0327.md).

## Why is this a problem?

Expanding a simple auto-property across multiple lines adds visual noise without conveying any extra information. A multi-line form like

```cs
public int Count
{
    get;
    set;
}
```

offers no readability advantage over the compact form `public int Count { get; set; }` and makes the code harder to scan quickly.

## How to fix it

Collapse the entire property declaration — type, identifier, and accessor list — onto a single line. If the declaration needs attributes or a wrapped signature to stay readable, leave it multi-line.

## Examples

### Violation

```cs
public int Count
{
    get;
    set;
}

public string Name
{
    get;
}
```

### Correction

```cs
public int Count { get; set; }

public string Name { get; }
```
