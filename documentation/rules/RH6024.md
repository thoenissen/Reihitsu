# RH6024 — Binary operators must be spaced correctly

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH6024       |
| **Category** | Spacing |
| **Severity** | Warning      |
| **Code Fix** | ✓            |

## Description

A binary operator must be separated from the token before it and the token after it by exactly one space. This covers the arithmetic (`+`, `-`, `*`, `/`, `%`), comparison (`==`, `!=`, `<`, `>`, `<=`, `>=`), logical (`&&`, `||`), bitwise (`&`, `|`, `^`), shift (`<<`, `>>`, `>>>`) and null-coalescing (`??`) operators.

The keyword operators `is` and `as` are governed by RH6005, and operators that wrap to a new line are left untouched.

## Why is this a problem?

Missing spaces (`a+b`) run the operands together and hurt readability. Extra spaces used to vertically align operators are fragile, produce noisy diffs, and are inconsistent with the single-space spacing enforced everywhere else.

## How to fix it

Use a single space on both sides of the operator. The code fix applies this change automatically.

## Examples

### Violation

```cs
internal class TestClass
{
    int Method(int a, int b)
    {
        return a  +  b;
    }
}
```

### Correction

```cs
internal class TestClass
{
    int Method(int a, int b)
    {
        return a + b;
    }
}
```
