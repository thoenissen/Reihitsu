# RH7103 — Static elements must appear before instance elements

| Property | Value |
|----------|-------|
| **ID** | RH7103 |
| **Category** | Organization |
| **Severity** | Warning |
| **Code Fix** | ✓ |

## Description

This rule requires static members to appear before instance members of the same kind and accessibility.

## Why is this a problem?

Placing static and instance members in a predictable order makes the public shape of a type easier to understand. Readers can quickly see the static API before they move on to instance behavior.

## How to fix it

Move static members above the matching instance members in the same group.

## Region scope

When a type uses top-level `#region` blocks, the ordering is evaluated within each region independently. Static members in one region are not compared against instance members in a different region, so per-region groupings such as `#region Lifecycle` and `#region Factories` can coexist without conflicting with this rule. A type without regions continues to be evaluated as a single body.

## Examples

### Violation

```cs
public class Example
{
    public void Run()
    {
    }

    public static void Create()
    {
    }
}
```

### Correction

```cs
public class Example
{
    public static void Create()
    {
    }

    public void Run()
    {
    }
}
```
