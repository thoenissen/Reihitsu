# RH0120 — Nested delegates should not be used

| Property | Value |
|----------|-------|
| **ID** | RH0120 |
| **Category** | Design |
| **Severity** | Warning |
| **Code Fix** | ❌ |

## Description

Do not declare a delegate inside another type.

## Why is this a problem?

Nested delegates are less discoverable and harder to reuse. Top-level delegates make call-site contracts easier to find and share.

## How to fix it

Move the nested delegate to a top-level declaration.

## Examples

### Violation

```cs
namespace Example;

internal class Container
{
    private delegate void Handler(int value);
}
```

### Correction

```cs
namespace Example;

internal class Container
{
}

internal delegate void Handler(int value);
```
