# RH5402 — Braces for multi-line statements must not share line

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH5402       |
| **Category** | Layout |
| **Severity** | Warning      |
| **Code Fix** | ✓            |

## Description

The opening brace of a multi-line block must appear on its own line, not on the same line as the preceding statement or declaration.

## Why is this a problem?

An opening brace that shares a line with the statement header makes it harder to see the block boundary at a glance and is inconsistent with the rest of the codebase.

## How to fix it

Move the opening brace to its own line immediately after the statement header. The code fix applies this change automatically.

## Examples

### Violation

```cs
internal class TestClass
{
    void Method()
    {
        if (true) {
            int value = 0;
        }
    }
}
```

### Correction

```cs
internal class TestClass
{
    void Method()
    {
        if (true)
        {
            int value = 0;
        }
    }
}
```
