# RH0224 — Tuple expression argument names should be in camelCase

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH0224       |
| **Category** | Naming       |
| **Severity** | Warning      |
| **Code Fix** | ✗            |

## Description

Checks that named arguments in tuple expressions follow camelCase naming conventions. The first word should start with a lowercase letter, and each subsequent word should start with an uppercase letter.

## Why is this a problem?

camelCase for tuple argument names is consistent with local variable and parameter naming conventions. Using PascalCase for tuple arguments creates inconsistency and can be confused with property access.

## How to fix it

Rename the tuple expression argument names to use camelCase.

## Examples

### Violation

```cs
var tuple = (Name: "test", Age: 5);
```

### Correction

```cs
var tuple = (name: "test", age: 5);
```
