# RH0102 — Async void methods should not be used

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH0102       |
| **Category** | Design       |
| **Severity** | Warning      |
| **Code Fix** | ✗            |

## Description

Async void methods should not be used. This rule flags methods declared as `async void` instead of `async Task` or `async Task<T>`.

## Why is this a problem?

Async void methods cannot be awaited, which means callers have no way to know when the operation completes. Exceptions thrown inside async void methods cannot be caught by the caller and will crash the process. They are also difficult to unit test reliably.

## How to fix it

Change the return type from `void` to `Task` or `Task<T>`.

## Examples

### Violation

```cs
private async void LoadDataAsync()
{
    await Task.Delay(1000);
}
```

### Correction

```cs
private async Task LoadDataAsync()
{
    await Task.Delay(1000);
}
```
