# RH0449 - XML documentation element text must not end with a period

| Property | Value |
|----------|-------|
| **ID** | RH0449 |
| **Category** | Documentation |
| **Severity** | Warning |
| **Code Fix** | ✓ |

## Description

This rule ensures that narrative XML documentation element text does not end with a trailing period.

## Why is this a problem?

Trailing periods make documentation inconsistent with the style expected by this repository. Removing them keeps summaries, parameter descriptions, and return descriptions visually uniform.

## How to fix it

Remove the final period from supported XML documentation element text such as `<summary>`, `<param>`, `<typeparam>`, `<returns>`, `<remarks>`, and similar descriptive elements.

## Examples

### Violation

```cs
internal class TestClass
{
    /// <summary>
    /// Validates the input.
    /// </summary>
    /// <param name="value">The value to validate.</param>
    /// <returns>True if the input is valid.</returns>
    bool Method(string value)
    {
        return value.Length > 0;
    }
}
```

### Correction

```cs
internal class TestClass
{
    /// <summary>
    /// Validates the input
    /// </summary>
    /// <param name="value">The value to validate</param>
    /// <returns>True if the input is valid</returns>
    bool Method(string value)
    {
        return value.Length > 0;
    }
}
```
