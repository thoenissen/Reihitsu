# RH6004 — Preprocessor keywords must not be preceded by space

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH6004       |
| **Category** | Spacing      |
| **Severity** | Warning      |
| **Code Fix** | ✓            |

## Description

Active non-region preprocessor directives (`#pragma`, `#if`, etc.) must start at column zero without any leading whitespace.

## Why is this a problem?

Indented preprocessor keywords look visually indented like code but are semantically at the file level. Leading whitespace creates confusion and inconsistency.

## How to fix it

Remove all leading whitespace before the `#` character. The code fix applies this change automatically.

## Examples

### Violation

```cs
    #pragma warning disable CS0168
internal class TestClass
{
}
```

### Correction

```cs
#pragma warning disable CS0168
internal class TestClass
{
}
```
