# RH0328 — #region description should start with an uppercase letter

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH0328       |
| **Category** | Formatting   |
| **Severity** | Warning      |
| **Code Fix** | ✔️            |

## Description

This rule checks that the description text following a `#region` directive starts with an uppercase letter. Consistent casing in region descriptions improves readability and maintains a professional code style.

## Why is this a problem?

Region descriptions that start with a lowercase letter are inconsistent with common C# naming conventions. Since regions are used to organize code sections, their descriptions should follow a consistent capitalization style to make file navigation easier.

## How to fix it

Capitalize the first letter of the `#region` description.

## Examples

### Violation

```cs
#region methods

public void DoWork()
{
}

#endregion // methods
```

### Correction

```cs
#region Methods

public void DoWork()
{
}

#endregion // Methods
```
