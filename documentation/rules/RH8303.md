# RH8303 — Element documentation header must be preceded by blank line

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH8303       |
| **Category** | Documentation |
| **Severity** | Warning      |
| **Code Fix** | ✓            |

## Description

A `///` documentation block that immediately follows another member must be preceded by a blank line.

## Why is this a problem?

Without a blank line before the documentation block, successive members and their documentation run together visually, making it harder to distinguish where one member ends and the next begins.

## How to fix it

Insert a blank line before the `///` documentation block. The code fix applies this change automatically.

## Exemptions

A documentation header that directly follows an ordinary comment (`//` or `////`) or a preprocessor directive is not flagged. The formatter treats adjacent comment blocks as a single unit and leaves directive-adjacent comments untouched, so these positions have no automatic remediation. This mirrors the exemptions applied by RH5020.

## Examples

### Violation

```cs
internal class TestClass
{
    void First()
    {
    }
    /// <summary>
    /// Summary.
    /// </summary>
    void Second()
    {
    }
}
```

### Correction

```cs
internal class TestClass
{
    void First()
    {
    }

    /// <summary>
    /// Summary.
    /// </summary>
    void Second()
    {
    }
}
```
