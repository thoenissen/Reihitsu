# RH0215 — Public field names should be in PascalCase

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH0215       |
| **Category** | Naming       |
| **Severity** | Warning      |
| **Code Fix** | ✓            |

## Description

Checks that public field names follow PascalCase naming conventions. Each word in the name should start with an uppercase letter, with no underscores or other separators.

## Why is this a problem?

PascalCase for public members is a fundamental C# naming convention. Inconsistent naming on public API surfaces confuses consumers and breaks established conventions.

## How to fix it

Rename the public field to use PascalCase.

## Examples

### Violation

```cs
public int myField;
```

### Correction

```cs
public int MyField;
```
