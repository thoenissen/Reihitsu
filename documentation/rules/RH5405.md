# RH5405 — Braces must not be omitted

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH5405       |
| **Category** | Layout |
| **Severity** | Warning      |
| **Code Fix** | ✓            |

## Description

A single-line child statement of an `if` or `else` must always use braces, even when the body is a single statement.

Multi-line brace-less child statements are reported by [RH5406](RH5406.md) instead, so the two rules never report the same statement.

## Why is this a problem?

Brace-less bodies are easy to misread and are a common source of bugs when additional statements are added later without noticing that the original body was not enclosed.

## How to fix it

Wrap the body of the control statement in braces. The code fix applies this change automatically.

## Examples

### Violation

```cs
internal class TestClass
{
    void Method()
    {
        if (true)
            return;
    }
}
```

### Correction

```cs
internal class TestClass
{
    void Method()
    {
        if (true)
        {
            return;
        }
    }
}
```
