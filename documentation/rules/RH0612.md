# RH0612 — Readonly elements must appear before non-readonly elements

| Property | Value |
|----------|-------|
| **ID** | RH0612 |
| **Category** | Ordering |
| **Severity** | Warning |
| **Code Fix** | ✓ |

## Description

This rule requires `readonly` fields to appear before mutable fields in the same group.

## Why is this a problem?

Placing immutable fields first makes it easier to see which parts of an object's state are fixed after construction. That improves readability when scanning a type.

## How to fix it

Move `readonly` fields above the matching non-readonly fields.

## Examples

### Violation

```cs
public class Example
{
    private int _value;
    private readonly int _maxValue;
}
```

### Correction

```cs
public class Example
{
    private readonly int _maxValue;
    private int _value;
}
```
