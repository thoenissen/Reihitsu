# RH5306 — List patterns should be formatted correctly

| Property | Value |
|----------|-------|
| **ID** | RH5306 |
| **Category** | Layout |
| **Severity** | Warning |
| **Code Fix** | ✓ |

## Description

This rule enforces consistent layout for C# list patterns (`[...]`), including slice variants such as `[.. var rest]`.

## Why is this a problem?

List patterns are often used in `is` expressions and `switch` arms. Inconsistent bracket alignment and mixed element line placement make matching logic harder to scan.

## How to fix it

Use one of these layouts:

1. Keep the list pattern on one line.
2. Use a multiline layout where opening and closing brackets are aligned and each inner pattern is on its own line.

The code fix reformats the surrounding code automatically using the formatter.

## Examples

### Violation

```cs
return values is [
    1, .. var remainig
          ];
```

### Correction

```cs
return values is [
                     1,
                     .. var remainig
                 ];
```
