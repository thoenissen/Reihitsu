# RH0209 — Method names should be in PascalCase

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH0209       |
| **Category** | Naming       |
| **Severity** | Warning      |
| **Code Fix** | ✓            |

## Description

Checks that method names follow PascalCase naming conventions. Each word in the name should start with an uppercase letter, with no underscores or other separators.

## Why is this a problem?

PascalCase for method names is a fundamental C# naming convention. Inconsistent casing reduces readability and makes it harder for developers to navigate and understand the codebase.

## How to fix it

Rename the method to use PascalCase.

## Examples

### Violation

```cs
public void doWork()
{
}
```

### Correction

```cs
public void DoWork()
{
}
```
