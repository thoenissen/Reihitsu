# RH0447 - XML documentation elements must be on separate lines

| Property | Value |
|----------|-------|
| **ID** | RH0447 |
| **Category** | Documentation |
| **Severity** | Warning |
| **Code Fix** | ✓ |

## Description

This rule ensures that adjacent XML documentation elements are written on separate lines.

## Why is this a problem?

When multiple documentation elements share a line, the structure of the comment becomes harder to scan. Splitting elements onto separate lines improves readability and keeps documentation diffs cleaner.

## How to fix it

Move each XML documentation element to its own `///` line. Keep inline elements such as `<see/>` inside text when they are part of a larger documentation element.

## Examples

### Violation

```cs
internal class TestClass
{
    /// <summary>
    /// Summary.
    /// </summary><param name="value">Value.</param>
    void Method(string value)
    {
    }
}
```

### Correction

```cs
internal class TestClass
{
    /// <summary>
    /// Summary.
    /// </summary>
    /// <param name="value">Value.</param>
    void Method(string value)
    {
    }
}
```
