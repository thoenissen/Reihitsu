# RH0382 — Do not combine fields

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH0382       |
| **Category** | Formatting   |
| **Severity** | Warning      |
| **Code Fix** | ✓            |

## Description

This rule checks whether a field declaration declares more than one variable.

## Why is this a problem?

Combining multiple fields into a single declaration makes field declarations harder to scan and harder to extend with individual comments or attributes later.

## How to fix it

Split the combined declaration into one field declaration per variable.

## Examples

### Violation

```cs
internal class TestClass
{
    private int firstField, secondField;
}
```

### Correction

```cs
internal class TestClass
{
    private int firstField;
    private int secondField;
}
```
