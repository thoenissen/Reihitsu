# RH0611 — Event accessors must follow order

| Property | Value |
|----------|-------|
| **ID** | RH0611 |
| **Category** | Ordering |
| **Severity** | Warning |
| **Code Fix** | ✓ |

## Description

This rule requires the `add` accessor to appear before `remove`.

## Why is this a problem?

Explicit event accessors are easier to read when they follow a consistent order. That consistency makes custom events look the same across the codebase.

## How to fix it

Move `add` above `remove`.

## Examples

### Violation

```cs
public event EventHandler Changed
{
    remove { _changed -= value; }
    add { _changed += value; }
}
```

### Correction

```cs
public event EventHandler Changed
{
    add { _changed += value; }
    remove { _changed -= value; }
}
```
