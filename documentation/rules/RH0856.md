# RH0856 - Accessor attribute lists must follow shape rules

| Property | Value |
|----------|-------|
| **ID** | RH0856 |
| **Category** | Formatting |
| **Severity** | Warning |
| **Code Fix** | Yes |

## Description

Accessor attribute-list shape depends on property layout:
- Multiline properties require split lists (`[A]` + `[B]`).
- Single-line auto-properties require merged lists (`[A, B]`).

## Examples

### Multiline violation

```cs
internal class Example
{
    internal int Value
    {
        [First, Second]
        get;
        set;
    }
}
```

### Multiline correction

```cs
internal class Example
{
    internal int Value
    {
        [First]
        [Second]
        get;
        set;
    }
}
```

### Single-line violation

```cs
internal class Example
{
    internal int Value { [First][Second] get; set; }
}
```

### Single-line correction

```cs
internal class Example
{
    internal int Value { [First, Second] get; set; }
}
```
