# RH0372 — File must not end with a newline

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH0372       |
| **Category** | Formatting   |
| **Severity** | Warning      |
| **Code Fix** | ✓            |

## Description

This rule requires a source file to end immediately after its last character instead of ending with an extra newline.

## Why is this a problem?

An extra newline at the end of a file adds avoidable formatting noise and makes file layout less predictable.

## How to fix it

Remove the final newline so the file ends with the last source character.

## Examples

### Violation

```cs
internal class TestClass
{
}

```

### Correction

```cs
internal class TestClass
{
}
```
