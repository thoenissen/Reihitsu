# RH0378 — Closing parenthesis must be on line of last argument

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH0378       |
| **Category** | Formatting   |
| **Severity** | Warning      |
| **Code Fix** | ✓            |

## Description

This rule ensures that methods and constructors place the closing parenthesis on the same line as the last argument.

## Why is this a problem?

Inconsistent formatting makes code harder to read and creates unnecessary noise in diffs.

## How to fix it

Adjust the affected formatting manually or apply the provided code fix.

## Examples

### Violation

```cs
internal class TestClass
{
    void Method1(int first, int second
                )
    {
    }

    void Method2(int first,
                int second
                )
    {
    }
}
```

### Correction

```cs
internal class TestClass
{
    void Method1(int first, int second)
    {
    }

    void Method2(int first,
                 int second)
    {
    }
}
```
