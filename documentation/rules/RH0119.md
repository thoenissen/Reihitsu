# RH0119 — Nested enums should not be used

| Property | Value |
|----------|-------|
| **ID** | RH0119 |
| **Category** | Design |
| **Severity** | Warning |
| **Code Fix** | ❌ |

## Description

Do not declare an enum inside another type.

## Why is this a problem?

Nested enums are less discoverable and tightly coupled to their containing type. Top-level enums are easier to share and maintain.

## How to fix it

Move the nested enum to a top-level declaration.

## Examples

### Violation

```cs
namespace Example;

internal class Container
{
    private enum Kind
    {
        One,
        Two,
    }
}
```

### Correction

```cs
namespace Example;

internal class Container
{
}

internal enum Kind
{
    One,
    Two,
}
```
