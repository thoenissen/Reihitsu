# RH3104 — Do not use default value type constructor

| Property | Value |
|----------|-------|
| **ID** | RH3104 |
| **Category** | Clarity |
| **Severity** | Warning |
| **Code Fix** | ✓ |

## Description

This rule discourages creating a value type with an empty constructor call such as `new Guid()` when the intent is just the default value.

## Why is this a problem?

`new T()` looks like a normal constructor call even though it only produces the default value for the struct. Using `default` makes that intent clear.

## How to fix it

Replace `new T()` with `default(T)` or `default` when appropriate.

## Exclusions

Structs that declare an explicit parameterless constructor are skipped, because `new S()` runs that constructor while `default(S)` does not.

Struct-constrained type parameters (`new T()` where `T : struct`) are skipped as well. The substituted struct may declare such a parameterless constructor, so `new T()` can run it while `default(T)` never does, and which substitution is used is unknown at analysis time.

## Examples

### Violation

```cs
var id = new Struct();
```

### Correction

```cs
var id = default(Struct);
```
