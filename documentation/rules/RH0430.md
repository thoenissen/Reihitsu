# RH0430 - Private events must be documented

| Property | Value |
|----------|-------|
| **ID** | RH0430 |
| **Category** | Documentation |
| **Severity** | Warning |
| **Code Fix** | ❌ |

## Description

This rule requires private event declarations to have XML documentation. A declaration counts as documented when it has a &lt;summary&gt; tag or uses &lt;inheritdoc/&gt;.

## Why is this a problem?

Undocumented private event declarations make the code harder to understand because readers have to infer intent from implementation details instead of reading a clear contract.

## How to fix it

Add a meaningful XML comment above the declaration. In most cases a short &lt;summary&gt; is the best fix.

## Examples

### Violation

```cs
/// <summary>
/// Publishes notifications
/// </summary>
internal class Publisher
{
    private event System.Action Changed;
}
```

### Correction

```cs
/// <summary>
/// Publishes notifications
/// </summary>
internal class Publisher
{
    /// <summary>
    /// Raised after the internal state changes
    /// </summary>
    private event System.Action Changed;
}
```
