# RH0605 — Protected must come before internal

| Property | Value |
|----------|-------|
| **ID** | RH0605 |
| **Category** | Ordering |
| **Severity** | Warning |
| **Code Fix** | ✓ |

## Description

This rule enforces the preferred keyword order for compound accessibility modifiers.

## Why is this a problem?

Compound access modifiers are easier to recognize when they always use the same wording. Inconsistent combinations such as `internal protected` create avoidable visual noise.

## How to fix it

Use `protected internal` instead of `internal protected`, and `private protected` instead of `protected private`.

## Examples

### Violation

```cs
public class Example
{
    internal protected void Execute()
    {
    }
}
```

### Correction

```cs
public class Example
{
    protected internal void Execute()
    {
    }
}
```
