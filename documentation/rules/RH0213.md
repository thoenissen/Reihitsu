# RH0213 — Protected field names should be in _camelCase

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH0213       |
| **Category** | Naming       |
| **Severity** | Warning      |
| **Code Fix** | ✓            |

## Description

Checks that protected field names follow _camelCase naming conventions with an underscore prefix. The first word after the underscore should start with a lowercase letter, and each subsequent word should start with an uppercase letter.

## Why is this a problem?

The underscore prefix convention for protected fields clearly distinguishes them from local variables, parameters, and public members. Consistent naming helps developers quickly identify the accessibility and scope of a field.

## How to fix it

Rename the protected field to use _camelCase with an underscore prefix.

## Examples

### Violation

```cs
protected int count;
```

### Correction

```cs
protected int _count;
```
