# RH0210 — Local function names should be in PascalCase

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH0210       |
| **Category** | Naming       |
| **Severity** | Warning      |
| **Code Fix** | ✓            |

## Description

Checks that local function names follow PascalCase naming conventions. Each word in the name should start with an uppercase letter, with no underscores or other separators.

## Why is this a problem?

PascalCase for function names is a standard C# naming convention that applies to local functions as well. Consistent naming improves readability and aligns local functions with regular method naming patterns.

## How to fix it

Rename the local function to use PascalCase.

## Examples

### Violation

```cs
public void Execute()
{
    void myLocal()
    {
    }
}
```

### Correction

```cs
public void Execute()
{
    void MyLocal()
    {
    }
}
```
