# RH0364 — Element documentation headers must not be followed by blank line

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH0364       |
| **Category** | Formatting   |
| **Severity** | Warning      |
| **Code Fix** | ✓            |

## Description

This rule ensures that element documentation headers must not be followed by blank line.

## Why is this a problem?

Inconsistent formatting makes code harder to read and creates unnecessary noise in diffs.

## How to fix it

Adjust the affected formatting manually or apply the provided code fix.

## Examples

### Violation

```cs
internal class TestClass
{
    /// <summary>
    /// Summary.
    /// </summary>

    void Method()
    {
    }
}
```

### Correction

```cs
internal class TestClass
{
    /// <summary>
    /// Summary.
    /// </summary>
    void Method()
    {
    }
}
```
