# RH0364 — Element documentation headers must not be followed by blank line

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH0364       |
| **Category** | Formatting   |
| **Severity** | Warning      |
| **Code Fix** | ✓            |

## Description

A `///` documentation block must not be separated from the member it documents by a blank line.

## Why is this a problem?

A blank line between the documentation and the member makes it harder to see at a glance that the comment belongs to the member directly below it.

## How to fix it

Remove the blank line between the documentation block and the member declaration. The code fix applies this change automatically.

## Examples

### Violation

```cs
internal class TestClass
{
    /// <summary>
    /// Summary.
    /// </summary>

    void Method()
    {
    }
}
```

### Correction

```cs
internal class TestClass
{
    /// <summary>
    /// Summary.
    /// </summary>
    void Method()
    {
    }
}
```
