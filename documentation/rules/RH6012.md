# RH6012 — Closing generic brackets must be spaced correctly

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH6012       |
| **Category** | Spacing |
| **Severity** | Warning      |
| **Code Fix** | ✓            |

## Description

No space may appear immediately before the closing angle bracket `>` of a generic type argument list.

## Why is this a problem?

A trailing space inside generic angle brackets is inconsistent with the rest of the language and makes generic types harder to recognize at a glance.

## How to fix it

Remove the space immediately before the closing generic angle bracket. The code fix applies this change automatically.

## Examples

### Violation

```cs
using System.Collections.Generic;
internal class TestClass
{
    void Method()
    {
        _ = new List<int >();
    }
}
```

### Correction

```cs
using System.Collections.Generic;
internal class TestClass
{
    void Method()
    {
        _ = new List<int>();
    }
}
```
