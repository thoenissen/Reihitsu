# RH0436 - Element parameter documentation must have text

| Property | Value |
|----------|-------|
| **ID** | RH0436 |
| **Category** | Documentation |
| **Severity** | Warning |
| **Code Fix** | ❌ |

## Description

This rule requires each `<param>` tag to contain descriptive text.

## Why is this a problem?

An empty parameter tag provides the shape of documentation without any information. That is confusing in IntelliSense and generated documentation.

## How to fix it

Write a short description for every documented parameter.

## Examples

### Violation

```cs
/// <summary>
/// Saves the customer
/// </summary>
/// <param name="customer"></param>
internal void Save(Customer customer)
{
}
```

### Correction

```cs
/// <summary>
/// Saves the customer
/// </summary>
/// <param name="customer">The customer to save</param>
internal void Save(Customer customer)
{
}
```
