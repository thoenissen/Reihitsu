# RH7305A — Types with multiple member kinds should be organized with regions

| Property | Value |
|----------|-------|
| **ID** | RH7305A |
| **Category** | Organization |
| **Severity** | Warning |
| **Enabled by default** | ❌ |
| **Code Fix** | ❌ |

## Description

This rule requires types that contain more than one kind of member to organize their direct members inside `#region` blocks. Types that only have one kind of member (for example, only properties or only methods) are not affected.

## Why is this a problem?

When a type mixes different kinds of members (such as fields, properties, constructors, and methods), region-based organization makes it significantly easier to navigate and locate members. For types with only a single member kind the benefit is smaller, so this alternative rule skips those cases.

## How to fix it

Wrap the direct members of the type in clearly named type-level regions. Each kind of member should be inside a dedicated top-level region such as `Fields`, `Properties`, `Constructors`, and `Methods`.

## Examples

### Violation

```cs
internal sealed class CustomerCache
{
    private readonly Dictionary<int, Customer> _cache;

    public CustomerCache()
    {
        _cache = new Dictionary<int, Customer>();
    }

    public int Count => _cache.Count;

    public Customer GetById(int id) => _cache[id];
}
```

### Correction

```cs
internal sealed class CustomerCache
{
    #region Fields

    private readonly Dictionary<int, Customer> _cache;

    #endregion // Fields

    #region Constructors

    public CustomerCache()
    {
        _cache = new Dictionary<int, Customer>();
    }

    #endregion // Constructors

    #region Properties

    public int Count => _cache.Count;

    #endregion // Properties

    #region Methods

    public Customer GetById(int id) => _cache[id];

    #endregion // Methods
}
```

### Not a violation (single member kind)

```cs
internal static class StringExtensions
{
    public static string Truncate(this string value, int maxLength)
    {
        return value.Length <= maxLength ? value : value[..maxLength];
    }

    public static bool IsNullOrEmpty(this string value)
    {
        return string.IsNullOrEmpty(value);
    }
}
```
