# RH2007 — Debug.Fail must provide message text

| Property | Value |
|----------|-------|
| **ID** | RH2007 |
| **Category** | Design |
| **Severity** | Warning |
| **Code Fix** | ❌ |

## Description

Calls to `Debug.Fail` must provide meaningful message text.

## Why is this a problem?

`Debug.Fail(null)`, `Debug.Fail("")`, or `Debug.Fail("   ")` give no clue why the failure path was reached.

## How to fix it

Pass a descriptive non-empty, non-whitespace message to `Debug.Fail`.

## Examples

### Violation

```cs
System.Diagnostics.Debug.Fail(null);
```

### Correction

```cs
System.Diagnostics.Debug.Fail("Reached unreachable branch in switch.");
```
