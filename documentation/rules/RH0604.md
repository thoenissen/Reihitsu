# RH0604 — Declaration keywords must follow order

| Property | Value |
|----------|-------|
| **ID** | RH0604 |
| **Category** | Ordering |
| **Severity** | Warning |
| **Code Fix** | ✓ |

## Description

This rule enforces a consistent order for declaration modifiers such as accessibility, `static`, `sealed`, `override`, `partial`, and `readonly`.

## Why is this a problem?

When modifiers appear in different orders, declarations become harder to skim and compare. A consistent sequence makes the important parts of a declaration easier to spot.

## How to fix it

Reorder the modifiers so accessibility comes first and the remaining modifiers follow the expected style.

## Examples

### Violation

```cs
public class Example
{
    static public int Value { get; set; }
}
```

### Correction

```cs
public class Example
{
    public static int Value { get; set; }
}
```
