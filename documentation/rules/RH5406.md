# RH5406 — Braces must not be omitted from multi-line child statements

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH5406       |
| **Category** | Layout |
| **Severity** | Warning      |
| **Code Fix** | ✓            |

## Description

A child statement of an `if` or `else` that spans multiple lines must always be enclosed in braces, even when only one logical statement is present.

Single-line brace-less child statements are reported by [RH5405](RH5405.md) instead, so the two rules never report the same statement.

## Why is this a problem?

A multi-line brace-less body is visually indistinguishable from a block body and is easy to misread, especially when the body contains a method call with multiple arguments split across lines.

## How to fix it

Wrap the multi-line child statement in braces. The code fix applies this change automatically.

## Examples

### Violation

```cs
internal class TestClass
{
    void Method()
    {
        if (true)
            Other(1);
    }
}
```

### Correction

```cs
internal class TestClass
{
    void Method()
    {
        if (true)
        {
            Other(1);
        }
    }
}
```
