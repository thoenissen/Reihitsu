# RH0003 — Code must not contain empty statements

| Property | Value |
|----------|-------|
| **ID** | RH0003 |
| **Category** | Clarity |
| **Severity** | Warning |
| **Code Fix** | ✓ |

## Description

This rule disallows standalone empty statements such as stray semicolons that do not perform any action.

## Why is this a problem?

Empty statements are usually leftovers from editing or refactoring. They add visual noise and can make readers wonder whether the statement was intentional.

## How to fix it

Remove the unnecessary empty statement.

## Examples

### Violation

```cs
public void Run()
{
    ;
    DoWork();
}
```

### Correction

```cs
public void Run()
{
    DoWork();
}
```
