# RH0381 — Parameters must be on same line or separate lines

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH0381       |
| **Category** | Formatting   |
| **Severity** | Warning      |
| **Code Fix** | ✓            |

## Description

This rule ensures that parameters must be on same line or separate lines.

## Why is this a problem?

Inconsistent formatting makes code harder to read and creates unnecessary noise in diffs.

## How to fix it

Adjust the affected formatting manually or apply the provided code fix.

## Examples

### Violation

```cs
internal class TestClass
{
    void Method(int first, int second,
                int third)
    {
    }
}
```

### Correction

```cs
internal class TestClass
{
    void Method(int first,
                int second,
                int third)
    {
    }
}
```
