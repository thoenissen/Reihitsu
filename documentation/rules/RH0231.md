# RH0231 — Record primary constructor parameter names should be in PascalCase

| Property | Value |
|----------|-------|
| **ID** | RH0231 |
| **Category** | Naming |
| **Severity** | Warning |
| **Code Fix** | ✓ |

## Description

Ensures primary-constructor parameters on `record` and `record struct` declarations use PascalCase.

## Why is this a problem?

This rule enforces a consistent style for record primary-constructor parameters. Mixed casing in these declarations makes naming conventions harder to follow.

## How to fix it

Rename each primary-constructor parameter so each word starts with an uppercase letter.

## Examples

### Violation

```cs
public record Product(string productCode);
```

### Correction

```cs
public record Product(string ProductCode);
```
