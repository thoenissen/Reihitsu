# RH0004 — Statement must not use unnecessary parentheses

| Property | Value |
|----------|-------|
| **ID** | RH0004 |
| **Category** | Clarity |
| **Severity** | Warning |
| **Code Fix** | ✓ |

## Description

This rule removes parentheses that do not change meaning and only make a statement harder to read.

## Why is this a problem?

Unnecessary parentheses create visual clutter. They can make simple expressions look more complex than they really are and reduce readability.

## How to fix it

Remove the parentheses when they do not affect the behavior of the expression.

## Examples

### Violation

```cs
public int GetValue(int value)
{
    return (value);
}
```

### Correction

```cs
public int GetValue(int value)
{
    return value;
}
```
