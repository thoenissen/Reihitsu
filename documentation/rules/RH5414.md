# RH5414 — Empty interfaces should use semicolon declarations

| Property | Value |
|----------|-------|
| **ID** | RH5414 |
| **Category** | Layout |
| **Severity** | Warning |
| **Code Fix** | ✓ |

## Description

This rule prefers semicolon declarations for empty interfaces.

Brace-based empty interface bodies add noise when the declaration contains no members and the language version supports semicolon syntax.

## Why is this a problem?

An empty interface already communicates everything through its declaration line. The extra braces do not add information.

Using a semicolon declaration makes empty contracts more concise and keeps declaration-only code easier to read.

## How to fix it

Replace the empty brace body with a trailing semicolon.

The code fix and formatter only apply this when the body is truly empty. Comments, directives, and other meaningful trivia inside the braces are preserved by leaving the declaration unchanged.

## Examples

### Violation

```cs
internal interface IExample
{
}
```

### Correction

```cs
internal interface IExample;
```
