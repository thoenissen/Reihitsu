# RH0433 - Element parameters must be documented

| Property | Value |
|----------|-------|
| **ID** | RH0433 |
| **Category** | Documentation |
| **Severity** | Warning |
| **Code Fix** | ❌ |

## Description

This rule requires a `<param>` tag for each parameter on a documented method, constructor, delegate, or indexer.

## Why is this a problem?

Parameters are part of the public contract of a declaration. If they are undocumented, callers have to inspect the implementation or rely on naming alone.

## How to fix it

Add a `<param>` tag for every parameter and describe what each one means.

## Examples

### Violation

```cs
/// <summary>
/// Saves the customer
/// </summary>
internal void Save(Customer customer, bool overwrite)
{
}
```

### Correction

```cs
/// <summary>
/// Saves the customer
/// </summary>
/// <param name="customer">The customer to save</param>
/// <param name="overwrite">Whether an existing customer may be replaced</param>
internal void Save(Customer customer, bool overwrite)
{
}
```
