# RH0002 — Do not prefix calls with `base.` unless a local implementation exists

| Property | Value |
|----------|-------|
| **ID** | RH0002 |
| **Category** | Clarity |
| **Severity** | Warning |
| **Code Fix** | ✓ |

## Description

This rule discourages using `base.` when the current type does not provide its own implementation of the accessed member. In that case the qualifier adds noise without clarifying behavior.

## Why is this a problem?

An unnecessary `base.` suggests that there is an override or local implementation being intentionally bypassed. If no such implementation exists, the qualifier is misleading and makes the code harder to read.

## How to fix it

Remove the unnecessary `base.` qualifier and call the member directly.

## Examples

### Violation

```cs
public class Derived : Base
{
    public string GetName()
    {
        return base.BuildName();
    }
}
```

### Correction

```cs
public class Derived : Base
{
    public string GetName()
    {
        return BuildName();
    }
}
```
