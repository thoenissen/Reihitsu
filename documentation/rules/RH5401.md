# RH5401 — Expression style get-only properties should be single lined

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH5401       |
| **Category** | Layout |
| **Severity** | Warning      |
| **Code Fix** | ✔️            |

## Description

This rule checks that expression-bodied get-only properties keep the `=>` arrow and the start of the expression on the property signature line. The expression itself may still wrap onto following lines (for example a switch expression, a wrapped fluent chain, or a multi-line collection expression); only the arrow and the first expression token are required to stay on the signature line.

## Why is this a problem?

Dropping the `=>` or the beginning of the expression onto a separate line defeats the purpose of using the concise expression syntax and is harder to read than keeping the expression anchored to the signature. Allowing the remainder of the expression to wrap keeps the rule aligned with the formatter, which only collapses the arrow and the first expression token onto the signature line.

## How to fix it

Move the `=>` arrow and the start of the expression onto the property signature line. Short expressions end up on a single line; longer expressions may continue on following lines.

## Examples

### Violation

```cs
public int Value
    => 42;
```

### Correction

```cs
public int Value => 42;
```
