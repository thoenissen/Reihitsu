# RH0115 — Structs should not use parameterized primary constructors

| Property | Value |
|----------|-------|
| **ID** | RH0115 |
| **Category** | Design |
| **Severity** | Warning |
| **Code Fix** | ❌ |

## Description

Disallows primary constructors with parameters on struct declarations.

## Why is this a problem?

Parameterized primary constructors on structs can blur where initialization happens and reduce consistency with explicit constructor patterns used in other types. Explicit constructors make intent clearer, especially for value types with multiple fields.

## How to fix it

Remove constructor parameters from the struct declaration and declare an explicit constructor inside the struct body.

## Examples

### Violation

```cs
namespace Example;

internal struct Dimensions(int width, int height)
{
}
```

### Correction

```cs
namespace Example;

internal struct Dimensions
{
    internal Dimensions(int width, int height)
    {
    }
}
```
