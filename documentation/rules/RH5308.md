# RH5308 — Conditional expressions should be formatted correctly

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH5308       |
| **Category** | Layout |
| **Severity** | Warning      |
| **Code Fix** | ✓            |

## Description

This rule ensures that a conditional (ternary) expression that spans multiple lines places its `?` and `:` tokens at the start of their own lines, indented consistently relative to the condition. A conditional that fits on a single line is never reported.

## Why is this a problem?

When a multi-line conditional keeps `?` or `:` at the end of a line or at an inconsistent indentation, the two branches are hard to line up visually and the structure of the expression is easy to misread. Anchoring the operators on their own indented lines makes the condition, the true branch, and the false branch immediately distinguishable when scanning the left edge of the code.

## How to fix it

Put each `?` and `:` at the beginning of its own line and indent it one level deeper than the condition. The code fix reformats the expression automatically.

## Examples

### Violation

```cs
var value = condition
? "1"
: "2";
```

### Correction

```cs
var value = condition
                ? "1"
                : "2";
```
