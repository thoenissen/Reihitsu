# RH0202 — Class names should be in PascalCase

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH0202       |
| **Category** | Naming       |
| **Severity** | Warning      |
| **Code Fix** | ✓            |

## Description

Checks that class names follow PascalCase naming conventions. Each word in the name should start with an uppercase letter, with no underscores or other separators.

## Why is this a problem?

PascalCase for type names is a fundamental C# naming convention. Inconsistent casing reduces readability and makes it harder for developers to identify types at a glance.

## How to fix it

Rename the class to use PascalCase.

## Examples

### Violation

```cs
public class myClass
{
}
```

### Correction

```cs
public class MyClass
{
}
```
