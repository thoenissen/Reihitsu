# RH0383 — Do not place regions within elements

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH0383       |
| **Category** | Formatting   |
| **Severity** | Warning      |
| **Code Fix** | ✓            |

## Description

This rule checks whether `#region` and `#endregion` directives are placed inside method bodies, accessor bodies, local functions, or other element bodies.

## Why is this a problem?

Regions inside executable code hide logic and make control flow harder to follow.

## How to fix it

Remove the region directives and keep the enclosed code in place.

## Examples

### Violation

```cs
internal class TestClass
{
    void Method()
    {
        #region Helper
        var value = 1;
        #endregion
    }
}
```

### Correction

```cs
internal class TestClass
{
    void Method()
    {
        var value = 1;
    }
}
```
