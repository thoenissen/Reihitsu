# RH6005 — Operator keyword must be followed by space

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH6005       |
| **Category** | Spacing |
| **Severity** | Warning      |
| **Code Fix** | ✓            |

## Description

The `operator` keyword must be followed by exactly one space before the operator symbol.

## Why is this a problem?

Without a space, `operator+` looks like a single identifier rather than a keyword followed by a symbol, making operator declarations harder to read.

## How to fix it

Insert a space between the `operator` keyword and the operator symbol. The code fix applies this change automatically.

## Examples

### Violation

```cs
internal class TestClass
{
    public static TestClass operator+(TestClass left, TestClass right) => left;
}
```

### Correction

```cs
internal class TestClass
{
    public static TestClass operator +(TestClass left, TestClass right) => left;
}
```
