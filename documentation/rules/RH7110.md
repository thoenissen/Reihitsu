# RH7110 — Members must be ordered by accessibility

| Property | Value |
|----------|-------|
| **ID** | RH7110 |
| **Category** | Organization |
| **Severity** | Warning |
| **Code Fix** | ✗ |

## Description

This rule requires members of a type to be ordered from most to least accessible. A member declared with a broader accessibility must not appear after a member of the same kind declared with a narrower accessibility. The order, from most to least accessible, is:

`public` → `internal` → `protected internal` → `protected` → `private protected` → `private`

## Why is this a problem?

Ordering members by accessibility keeps the most visible surface of a type at the top, so readers see the public API before the implementation details. Interleaving `private`, `protected`, and `public` members forces the reader to scan the whole body to understand what a type exposes.

## How to fix it

Move members so that, within each kind, broader-accessibility members appear before narrower-accessibility ones.

## Scope

The ordering is evaluated per member kind. Members of different kinds (for example fields and methods) are not compared against each other, so this rule does not conflict with the kind-based grouping enforced by the region rules (RH7305) or with the static, constant, and readonly ordering rules (RH7103, RH7102, RH7109), which each apply within a single accessibility group. Combined with those rules, the canonical order within a kind becomes: accessibility, then static before instance, then constant before field, then readonly before non-readonly.

Members without an explicit accessibility modifier are ignored, because their effective accessibility depends on the declaring type. RH7104 governs missing modifiers separately.

## Region scope

When a type uses top-level `#region` blocks, the ordering is evaluated within each region independently. A member in one region is not compared against a member in a different region, so per-region groupings can coexist with this rule. A type without regions is evaluated as a single body.

## Examples

### Violation

```cs
public class Example
{
    private void Helper()
    {
    }

    public void Run()
    {
    }
}
```

### Correction

```cs
public class Example
{
    public void Run()
    {
    }

    private void Helper()
    {
    }
}
```
