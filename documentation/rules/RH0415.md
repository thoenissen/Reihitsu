# RH0415 - Enum members in private enums must be documented

| Property | Value |
|----------|-------|
| **ID** | RH0415 |
| **Category** | Documentation |
| **Severity** | Warning |
| **Code Fix** | ❌ |

## Description

This rule requires enum members in private enums to have XML documentation. A member counts as documented when it has a `<summary>` tag or uses `<inheritdoc/>`.

## Why is this a problem?

Undocumented enum members in private enums make internal implementation details harder to understand and maintain. Readers have to infer what the values mean instead of reading a clear explanation.

## How to fix it

Add a meaningful XML comment above each enum member in the private enum. In most cases a short `<summary>` is the best fix.

## Examples

### Violation

```cs
internal class Container
{
    private enum OrderState
    {
        Open,
    }
}
```

### Correction

```cs
internal class Container
{
    private enum OrderState
    {
        /// <summary>
        /// The order was created and is waiting for processing
        /// </summary>
        Open,
    }
}
```
