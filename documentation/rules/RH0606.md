# RH0606 — System using directives must be placed before other using directives

| Property | Value |
|----------|-------|
| **ID** | RH0606 |
| **Category** | Ordering |
| **Severity** | Warning |
| **Code Fix** | ✓ |

## Description

This rule requires `System` namespace imports to appear before other regular `using` directives.

## Why is this a problem?

Keeping framework imports together at the top makes the using block easier to scan. It also creates a predictable structure across files.

## How to fix it

Move `using System...` directives above other regular `using` directives.

## Examples

### Violation

```cs
using MyProject.Common;
using System.Linq;
```

### Correction

```cs
using System.Linq;
using MyProject.Common;
```
