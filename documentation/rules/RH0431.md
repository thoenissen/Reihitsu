# RH0431 - Element documentation must have summary text

| Property | Value |
|----------|-------|
| **ID** | RH0431 |
| **Category** | Documentation |
| **Severity** | Warning |
| **Code Fix** | ❌ |

## Description

This rule requires every &lt;summary&gt; tag to contain meaningful text.

## Why is this a problem?

An empty summary is effectively the same as no summary at all. It adds XML noise without helping readers understand the declaration.

## How to fix it

Write a short, useful description inside the &lt;summary&gt; tag.

## Examples

### Violation

```cs
/// <summary></summary>
internal partial class Worker
{
}
```

### Correction

```cs
/// <summary>
/// Performs background work for the queue
/// </summary>
internal partial class Worker
{
}
```
