# RH0333 — Multi-line arguments should be aligned

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH0333       |
| **Category** | Formatting   |
| **Severity** | Warning      |
| **Code Fix** | ✓            |

## Description

This rule checks that when arguments in a method or constructor call span multiple lines, each argument is aligned to the column immediately after the opening parenthesis. Consistent alignment makes it easy to see where each argument begins.

## Why is this a problem?

Misaligned arguments in multi-line calls make it difficult to visually associate each argument with the call. Consistent column alignment provides a clear structure.

## How to fix it

Align each argument to the column right after the opening parenthesis. The code fix will automatically adjust the indentation of each misaligned argument.

## Examples

### Violation

```cs
a.Call("test1",
  "test2",
  "test3");
```

### Correction

```cs
a.Call("test1",
       "test2",
       "test3");
```
