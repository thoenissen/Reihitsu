# RH0442 - Generic type parameter documentation must declare parameter name

| Property | Value |
|----------|-------|
| **ID** | RH0442 |
| **Category** | Documentation |
| **Severity** | Warning |
| **Code Fix** | ❌ |

## Description

This rule requires each `<typeparam>` tag to declare the generic type parameter name through its `name` attribute.

## Why is this a problem?

Without the type parameter name, tooling and readers cannot tell which generic parameter the documentation describes.

## How to fix it

Add the correct `name` attribute to every `<typeparam>` tag.

## Examples

### Violation

```cs
/// <summary>
/// Represents a repository
/// </summary>
/// <typeparam>The entity type stored by the repository</typeparam>
internal class Repository<T>
{
}
```

### Correction

```cs
/// <summary>
/// Represents a repository
/// </summary>
/// <typeparam name="T">The entity type stored by the repository</typeparam>
internal class Repository<T>
{
}
```
