# RH5411 — Final collection initializer items must not have trailing commas

| Property | Value |
|----------|-------|
| **ID** | RH5411 |
| **Category** | Layout |
| **Severity** | Warning |
| **Code Fix** | ✓ |

## Description

This rule forbids a trailing comma after the final item in a collection initializer.

## Why is this a problem?

A trailing comma on the last collection item adds noise without improving readability.

Removing it keeps collection initializers visually consistent and makes the end of the initializer easier to scan.

## How to fix it

Remove the comma after the final collection item. If the item has an attached comment, keep the comment with the item and remove only the comma.

The code fix and formatter can apply this automatically.

## Examples

### Violation

```cs
using System.Collections.Generic;

internal class Example
{
    private static readonly List<int> Values = new()
                                          {
                                              1,
                                              2, // final value
                                          };
}
```

### Correction

```cs
using System.Collections.Generic;

internal class Example
{
    private static readonly List<int> Values = new()
                                          {
                                              1,
                                              2 // final value
                                          };
}
```
