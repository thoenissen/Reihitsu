# RH5107 — Comma must be on same line as previous parameter

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH5107       |
| **Category** | Layout |
| **Severity** | Warning      |
| **Code Fix** | ✓            |

## Description

In a multi-line parameter or argument list, the comma separating elements must appear at the end of the line that contains the preceding element, not at the start of the following line.

## Why is this a problem?

A leading comma on the next line is non-standard in C# and makes it harder to read parameter lists by moving the separator away from the element it follows.

## How to fix it

Move the comma to the end of the preceding line. The code fix applies this change automatically.

## Examples

### Violation

```cs
internal class TestClass
{
    void Method(int first
               ,int second)
    {
    }
}
```

### Correction

```cs
internal class TestClass
{
    void Method(int first,
                int second)
    {
    }
}
```
