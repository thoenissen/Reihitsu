# RH7107 — Property accessors must follow order

| Property | Value |
|----------|-------|
| **ID** | RH7107 |
| **Category** | Organization |
| **Severity** | Warning |
| **Code Fix** | ✓ |

## Description

This rule requires the `get` accessor to appear before `set` or `init`.

## Why is this a problem?

Property declarations are easier to read when accessors appear in a familiar order. Putting `get` first matches common C# style and improves consistency.

## How to fix it

Move `get` above `set` or `init`.

## Examples

### Violation

```cs
public string Name
{
    set { }
    get { return string.Empty; }
}
```

### Correction

```cs
public string Name
{
    get { return string.Empty; }
    set { }
}
```
