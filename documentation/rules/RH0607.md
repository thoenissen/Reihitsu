# RH0607 — Using alias directives must be placed after other using directives

| Property | Value |
|----------|-------|
| **ID** | RH0607 |
| **Category** | Ordering |
| **Severity** | Warning |
| **Code Fix** | ✓ |

## Description

This rule requires using-alias directives to appear after regular namespace imports.

## Why is this a problem?

Aliases are special-case imports. Keeping them grouped below ordinary namespace imports makes the file header easier to read.

## How to fix it

Move `using Alias = ...;` directives below the regular `using` directives.

## Examples

### Violation

```cs
using TextAlias = System.String;
using MyProject.Common;
```

### Correction

```cs
using MyProject.Common;
using TextAlias = System.String;
```
