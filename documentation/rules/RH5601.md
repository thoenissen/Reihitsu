# RH5601 — Tab characters must not be used

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH5601       |
| **Category** | Layout |
| **Severity** | Warning      |
| **Code Fix** | ✓            |

## Description

Tab characters must not appear in source code outside of string, character, interpolated-string, raw-string, and UTF-8 string literals. All indentation and horizontal spacing must use spaces. Tabs inside literal content are treated as semantic content and are left untouched.

## Why is this a problem?

Tabs render differently depending on the editor or tool. Mixing tabs and spaces causes inconsistent indentation across environments and is a common source of diff noise.

## How to fix it

Replace each tab character with the appropriate number of spaces. The code fix substitutes the tab automatically.

## Examples

### Violation

```cs
internal class TestClass
{
	void Method()
	{
	}
}
```

### Correction

```cs
internal class TestClass
{
    void Method()
    {
    }
}
```
