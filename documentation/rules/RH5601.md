# RH5601 — Tab characters must not be used

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH5601       |
| **Category** | Layout |
| **Severity** | Warning      |
| **Code Fix** | ✓            |

## Description

Tab characters must not appear in source code outside of string, character, interpolated-string, raw-string, and UTF-8 string literals. All indentation and horizontal spacing must use spaces. Tabs inside literal content are treated as semantic content and are left untouched. Tabs inside comments (`//`, `/* */`, `///`, `/** */`) and inside preprocessor-disabled text (for example the body of an `#if false` block) are exempt for the same reason: that content may be semantically meaningful — such as aligned example output, or code deliberately preserved inactive — and the formatter never rewrites it.

## Why is this a problem?

Tabs render differently depending on the editor or tool. Mixing tabs and spaces causes inconsistent indentation across environments and is a common source of diff noise.

## How to fix it

Replace each tab character with the appropriate number of spaces. The code fix substitutes the tab automatically, and running `reihitsu-format` normalizes any remaining tabs outside of literals, comments, and disabled text.

## Examples

### Violation

```cs
internal class TestClass
{
	void Method()
	{
	}
}
```

### Correction

```cs
internal class TestClass
{
    void Method()
    {
    }
}
```
