# RH0386 — Region directives must use consistent indentation with containing code

| Property | Value |
|----------|-------|
| **ID** | RH0386 |
| **Category** | Formatting |
| **Severity** | Warning |
| **Code Fix** | ✓ |

## Description

This rule requires `#region` and `#endregion` directives to be indented to the same level as the code they organize.

## Why is this a problem?

Misaligned region directives make the surrounding structure harder to read. When region markers do not line up with the members or types they contain, the nesting level of the code becomes less obvious.

## How to fix it

Indent each `#region` and `#endregion` directive to the same column as the members or declarations inside that region. Apply the provided code fix to normalize the indentation automatically.

## Examples

### Violation

```cs
internal class Example
{
#region Fields

    private string _name;

#endregion // Fields
}
```

### Correction

```cs
internal class Example
{
    #region Fields

    private string _name;

    #endregion // Fields
}
```
