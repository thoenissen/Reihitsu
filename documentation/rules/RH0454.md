# RH0454 - Extension declaration parameters must be documented

| Property | Value |
|----------|-------|
| **ID** | RH0454 |
| **Category** | Documentation |
| **Severity** | Warning |
| **Code Fix** | ❌ |

## Description

This rule requires each parameter declared on an `extension(...)` or `extension<T>(...)` declaration to have matching `<param>` documentation.

## Why is this a problem?

Extension declaration parameters define the receiver and contextual inputs for the whole extension block. Missing parameter documentation makes the extension surface harder to understand and use correctly.

## How to fix it

Add `<param>` tags for all extension declaration parameters and keep names aligned with the declaration.

## Examples

### Violation

```cs
public static class Extensions
{
    /// <summary>Provides text-related extensions.</summary>
    extension(string value)
    {
    }
}
```

### Correction

```cs
public static class Extensions
{
    /// <summary>Provides text-related extensions.</summary>
    /// <param name="value">The source text.</param>
    extension(string value)
    {
    }
}
```
