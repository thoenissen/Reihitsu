# RH0309 — for-Statement should be preceded by a blank line

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH0309       |
| **Category** | Formatting   |
| **Severity** | Warning      |
| **Code Fix** | ✔️            |

## Description

This rule checks that `for` statements are preceded by a blank line. The rule does not apply when the `for` statement is the first statement in a block.

## Why is this a problem?

Separating statements with blank lines improves code readability by visually grouping related logic. A blank line before a `for` loop clearly delineates the loop from preceding initialization or setup code.

## How to fix it

Add a blank line before the `for` statement. The rule does not apply when the statement is the first statement in a block.

## Examples

### Violation

```cs
var count = GetCount();
for (var i = 0; i < count; i++)
{
    Process(i);
}
```

### Correction

```cs
var count = GetCount();

for (var i = 0; i < count; i++)
{
    Process(i);
}
```
