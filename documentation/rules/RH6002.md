# RH6002 — Commas must be spaced correctly

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH6002       |
| **Category** | Spacing |
| **Severity** | Warning      |
| **Code Fix** | ✓            |

## Description

A comma must be followed by exactly one space and must not be preceded by a space.

## Why is this a problem?

Commas without a trailing space compress parameter or argument lists and make them harder to scan. A leading space before a comma is non-standard and distracting.

## How to fix it

Remove any space before the comma and ensure exactly one space follows it. The code fix applies this change automatically.

## Exceptions

Commas that do not separate values are left compact and are not reported:

- Array rank specifiers, such as `int[,]`.
- Unbound generic types where the type arguments are omitted, such as `typeof(Dictionary<,>)` or `typeof(Func<,,>)`.

## Examples

### Violation

```cs
internal class TestClass
{
    void Method(int x,int y)
    {
    }
}
```

### Correction

```cs
internal class TestClass
{
    void Method(int x, int y)
    {
    }
}
```
