# RH7411 — Interface implementation events should be grouped by interface regions

| Property | Value |
|----------|-------|
| **ID** | RH7411 |
| **Category** | Organization |
| **Severity** | Warning |
| **Code Fix** | ❌ |

## Description

This rule requires each event that implements an interface event to appear inside a top-level `#region` whose label matches the interface that declares the member.

## Why is this a problem?

Members implementing an interface are easier to scan when they are grouped by the interface they belong to. Generic region names such as `Events` hide that relationship and make types that implement several interfaces harder to navigate.

## How to fix it

Move each interface implementation event into a top-level region named after the implemented interface. Use the interface name only, such as `INotifyPropertyChanged`. This applies to both explicit and implicit implementations.

## Examples

### Violation

```cs
internal class TestClass : INotifyPropertyChanged
{
    #region Events

    public event PropertyChangedEventHandler PropertyChanged;

    #endregion // Events
}
```

### Correction

```cs
internal class TestClass : INotifyPropertyChanged
{
    #region INotifyPropertyChanged

    public event PropertyChangedEventHandler PropertyChanged;

    #endregion // INotifyPropertyChanged
}
```
