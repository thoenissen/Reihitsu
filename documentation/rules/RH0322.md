# RH0322 — Single line comments should be preceded by a blank line

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH0322       |
| **Category** | Formatting   |
| **Severity** | Warning      |
| **Code Fix** | ✓            |

## Description

This rule checks that single-line comments (`//`) are preceded by a blank line.

## Why is this a problem?

Single-line comments that are not separated from the preceding code by a blank line can appear cluttered and are easy to overlook. A blank line before a comment improves readability by visually associating the comment with the code that follows it rather than the code above.

## How to fix it

Add a blank line before the single-line comment.

## Examples

### Violation

```cs
var x = Calculate();
// Process the result
Process(x);
```

### Correction

```cs
var x = Calculate();

// Process the result
Process(x);
```
