# RH0225 — File scoped namespace declarations should be in PascalCase

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH0225       |
| **Category** | Naming       |
| **Severity** | Warning      |
| **Code Fix** | ✗            |

## Description

Checks that file-scoped namespace declarations follow PascalCase naming conventions. Each segment of the namespace should start with an uppercase letter, with no underscores or other separators.

## Why is this a problem?

PascalCase for namespaces is a fundamental C# naming convention. Inconsistent namespace casing makes it harder to reference types and breaks expectations across the .NET ecosystem.

## How to fix it

Rename the namespace segments to use PascalCase.

## Examples

### Violation

```cs
namespace my.app;
```

### Correction

```cs
namespace My.App;
```
