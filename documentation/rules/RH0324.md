# RH0324 — Method chains should be aligned

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH0324       |
| **Category** | Formatting   |
| **Severity** | Warning      |
| **Code Fix** | ✓            |

## Description

This rule checks that multi-line method chains have their dots aligned with the dot of the first method call. When a method chain spans multiple lines, each continuation line's `.` should be vertically aligned for consistent readability.

## Why is this a problem?

Misaligned method chains make it difficult to follow the sequence of operations, especially when chains are long. Consistent dot alignment provides a clear visual guide through the chain of calls.

## How to fix it

Align the leading `.` on each continuation line with the `.` on the first line of the chain. The code fix will automatically reformat the method chain.

## Examples

### Violation

```cs
var result = list.Where(x => x > 0)
.Select(x => x * 2)
.ToList();
```

### Correction

```cs
var result = list.Where(x => x > 0)
                 .Select(x => x * 2)
                 .ToList();
```
