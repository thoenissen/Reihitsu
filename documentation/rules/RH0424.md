# RH0424 - Private properties must be documented

| Property | Value |
|----------|-------|
| **ID** | RH0424 |
| **Category** | Documentation |
| **Severity** | Warning |
| **Code Fix** | ❌ |

## Description

This rule requires private property declarations to have XML documentation. A declaration counts as documented when it has a &lt;summary&gt; tag or uses &lt;inheritdoc/&gt;.

## Why is this a problem?

Undocumented private property declarations make the code harder to understand because readers have to infer intent from implementation details instead of reading a clear contract.

## How to fix it

Add a meaningful XML comment above the declaration. In most cases a short &lt;summary&gt; is the best fix.

## Examples

### Violation

```cs
/// <summary>
/// Stores application settings
/// </summary>
internal class Settings
{
    private string Name { get; }
}
```

### Correction

```cs
/// <summary>
/// Stores application settings
/// </summary>
internal class Settings
{
    /// <summary>
    /// Gets the cached profile name used by the settings loader
    /// </summary>
    private string Name { get; }
}
```
