# RH7405 — Non-override methods should not be placed in override regions

| Property | Value |
|----------|-------|
| **ID** | RH7405 |
| **Category** | Organization |
| **Severity** | Warning |
| **Code Fix** | ❌ |

## Description

This rule forbids non-override methods from being placed inside a top-level override region whose label is used by override methods in the same type.

## Why is this a problem?

Override regions are meant to group inherited behavior by the type that originally introduced it. Placing helper or utility methods inside those regions hides that intent and makes inheritance-focused navigation less reliable.

## How to fix it

Move each non-override method out of the override region and into a normal organizational region such as `Methods`, or leave it ungrouped if that matches the surrounding style.

## Examples

### Violation

```cs
internal abstract class BaseProcessor
{
    public virtual void Execute()
    {
    }
}

internal sealed class ReportProcessor : BaseProcessor
{
    #region BaseProcessor

    public override void Execute()
    {
    }

    private static string BuildCacheKey()
    {
        return "report";
    }

    #endregion // BaseProcessor
}
```

### Correction

```cs
internal abstract class BaseProcessor
{
    public virtual void Execute()
    {
    }
}

internal sealed class ReportProcessor : BaseProcessor
{
    #region Methods

    private static string BuildCacheKey()
    {
        return "report";
    }

    #endregion // Methods

    #region BaseProcessor

    public override void Execute()
    {
    }

    #endregion // BaseProcessor
}
```
