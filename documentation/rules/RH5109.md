# RH5109 — Parameters must be on same line or separate lines

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH5109       |
| **Category** | Layout |
| **Severity** | Warning      |
| **Code Fix** | ✓            |

## Description

In a parameter list, all parameters must either fit on one line or each parameter must be on its own line. Once the list spans multiple lines — including when a single parameter's type wraps across lines — every parameter must start on its own line. Mixing parameters that share a line with parameters on separate lines is not allowed.

## Why is this a problem?

Partially split parameter lists are harder to scan because the reader cannot quickly determine how many parameters there are or align them visually.

## How to fix it

Either keep all parameters on one line or put each parameter on its own line. The code fix will apply the separate-line form automatically.

## Examples

### Violation

```cs
internal class TestClass
{
    void Method(int first, int second,
                int third)
    {
    }
}
```

### Correction

```cs
internal class TestClass
{
    void Method(int first,
                int second,
                int third)
    {
    }
}
```
