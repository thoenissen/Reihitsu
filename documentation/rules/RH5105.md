# RH5105 — Opening parenthesis must be on declaration line

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH5105       |
| **Category** | Layout |
| **Severity** | Warning      |
| **Code Fix** | ✓            |

## Description

The opening parenthesis of a method or constructor parameter list must appear on the same line as the method or constructor name.

## Why is this a problem?

Moving the opening parenthesis to the next line separates the name from its parameter list, making the declaration harder to read and navigate.

## How to fix it

Move the opening parenthesis immediately after the method or constructor name on the same line. The code fix applies this change automatically.

## Examples

### Violation

```cs
internal class TestClass
{
    void Method
    (int value)
    {
    }
}
```

### Correction

```cs
internal class TestClass
{
    void Method(int value)
    {
    }
}
```
