# RH5105 — Opening parenthesis must be on declaration line

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH5105       |
| **Category** | Layout |
| **Severity** | Warning      |
| **Code Fix** | ✓            |

## Description

The opening parenthesis of a parameter list must appear on the same line as the preceding declaration token. This applies to methods, constructors, local functions, operators, conversion operators, delegate declarations, and record (and record struct) primary constructors.

## Why is this a problem?

Moving the opening parenthesis to the next line separates the declaration name from its parameter list, making the declaration harder to read and navigate.

## How to fix it

Move the opening parenthesis immediately after the preceding declaration token on the same line. The code fix applies this change automatically.

When a comment or directive sits in the gap between the declaration token and the opening parenthesis, the shape is not reported, because the formatter refuses to collapse across that trivia.

## Examples

### Violation

```cs
internal class TestClass
{
    void Method
    (int value)
    {
    }
}
```

### Correction

```cs
internal class TestClass
{
    void Method(int value)
    {
    }
}
```
