# RH0011 — Conditional expressions must declare precedence

| Property | Value |
|----------|-------|
| **ID** | RH0011 |
| **Category** | Clarity |
| **Severity** | Warning |
| **Code Fix** | ✓ |

## Description

This rule requires explicit parentheses when a condition mixes `&&` and `||`.

## Why is this a problem?

Even though C# defines the precedence of these operators, readers should not have to remember it while reading a condition. Parentheses make the grouping obvious.

## How to fix it

Wrap the intended sub-expression in parentheses.

## Examples

### Violation

```cs
if (isAdmin || isOwner && isActive)
{
}
```

### Correction

```cs
if (isAdmin || (isOwner && isActive))
{
}
```
