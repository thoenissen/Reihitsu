# RH0314 — continue-Statement should be preceded by a blank line

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH0314       |
| **Category** | Formatting   |
| **Severity** | Warning      |
| **Code Fix** | ✗            |

## Description

This rule checks that `continue` statements are preceded by a blank line. The rule does not apply when the `continue` statement is the first statement in a block.

## Why is this a problem?

Separating statements with blank lines improves code readability by visually grouping related logic. A blank line before a `continue` statement clearly indicates that the current loop iteration is about to be skipped.

## How to fix it

Add a blank line before the `continue` statement. The rule does not apply when the statement is the first statement in a block.

## Examples

### Violation

```cs
foreach (var item in items)
{
    var isValid = Validate(item);
    continue;
}
```

### Correction

```cs
foreach (var item in items)
{
    var isValid = Validate(item);

    continue;
}
```
