# RH8306 — XML documentation element text must not end with a period

| Property | Value |
|----------|-------|
| **ID** | RH8306 |
| **Category** | Documentation |
| **Severity** | Warning |
| **Code Fix** | ✓ |

## Description

This rule ensures that narrative single-sentence XML documentation element text does not end with a trailing period.

The rule applies only to text that consists of a single sentence. When the text contains more than one sentence, the trailing period is required for consistency with the earlier sentences and the rule does not fire.

## Why is this a problem?

Trailing periods make single-sentence documentation inconsistent with the style expected by this repository. Removing them keeps summaries, parameter descriptions, and return descriptions visually uniform.

Multi-sentence text is treated differently: stripping only the final period would leave every sentence except the last terminated, which is grammatically inconsistent and harder to read. Such text is left untouched.

## Multi-sentence detection

The text is considered to contain more than one sentence when an internal sentence terminator (`.`, `!`, or `?`) is followed by whitespace and an uppercase letter — the start of a further sentence. This heuristic keeps abbreviations (for example `e.g.`) and decimal numbers (for example `1.0`) inside a single sentence from being misclassified.

## How to fix it

Remove the final period from supported single-sentence XML documentation element text such as `<summary>`, `<param>`, `<typeparam>`, `<returns>`, `<remarks>`, and similar descriptive elements.

## Examples

### Violation

```cs
internal class TestClass
{
    /// <summary>
    /// Validates the input.
    /// </summary>
    /// <param name="value">The value to validate.</param>
    /// <returns>True if the input is valid.</returns>
    bool Method(string value)
    {
        return value.Length > 0;
    }
}
```

### Correction

```cs
internal class TestClass
{
    /// <summary>
    /// Validates the input
    /// </summary>
    /// <param name="value">The value to validate</param>
    /// <returns>True if the input is valid</returns>
    bool Method(string value)
    {
        return value.Length > 0;
    }
}
```

### Allowed (multi-sentence)

```cs
internal class TestClass
{
    /// <summary>
    /// Validates the input. Throws when the value is empty.
    /// </summary>
    void Method(string value)
    {
    }
}
```
