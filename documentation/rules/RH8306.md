# RH8306 — Property summary must use a noun phrase instead of a sentence

| Property | Value |
|----------|-------|
| **ID** | RH8306 |
| **Category** | Documentation |
| **Severity** | Warning |
| **Code Fix** | ✗ |

## Description

This rule ensures that the `<summary>` of a property is written as a short noun phrase (a heading-style label) rather than as a full sentence.

A property describes a value, so its summary should name that value (for example `Current project configuration`) instead of narrating behavior (for example `Gets or sets the current project configuration.`).

The rule applies only to the `<summary>` element of properties. Other documentation elements and other member kinds are not covered.

## Why is this a problem?

Property summaries read best as concise labels. A behavior-narrating sentence such as "Gets or sets the current project configuration." adds boilerplate ("Gets or sets") and a trailing period that carry no additional meaning for a property. Keeping property summaries as noun phrases makes generated documentation and IntelliSense uniform and easy to scan.

## Detection

A property `<summary>` is treated as a sentence, and therefore reported, when either of the following holds:

- the text ends with a sentence terminator (`.`, `!`, or `?`), or
- the text begins with a verb that introduces a behavior-narrating sentence: `Gets`, `Sets`, `Returns`, `Represents`, `Indicates`, `Specifies`, `Determines`, `Holds`, or `Stores` (the leading verb `Gets` also covers the common `Gets or sets` phrasing).

An empty summary and a property that inherits its documentation with `<inheritdoc/>` are not reported by this rule.

## How to fix it

Rewrite the property summary as a noun phrase: drop any leading verb such as "Gets or sets" and remove the trailing terminator. Because this rewrite cannot be produced automatically without changing the wording, the rule does not offer a code fix.

## Examples

### Violation

```cs
internal class ProjectContext
{
    /// <summary>
    /// Gets or sets the current project configuration.
    /// </summary>
    public ProjectConfiguration Configuration { get; set; }
}
```

### Correction

```cs
internal class ProjectContext
{
    /// <summary>
    /// Current project configuration
    /// </summary>
    public ProjectConfiguration Configuration { get; set; }
}
```
