# RH0374 — Use braces consistently

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH0374       |
| **Category** | Formatting   |
| **Severity** | Warning      |
| **Code Fix** | ✓            |

## Description

When one branch of an `if`-`else` statement uses braces, all other branches must also use braces.

## Why is this a problem?

Mixing braced and brace-less branches in the same `if`-`else` chain is inconsistent and makes it easy to add statements to the wrong branch by misjudging which clause the indentation belongs to.

## How to fix it

Add braces to all branches so every clause uses them consistently. The code fix applies this change automatically.

## Examples

### Violation

```cs
internal class TestClass
{
    void Method()
    {
        if (true)
        {
            return;
        }
        else
            return;
    }
}
```

### Correction

```cs
internal class TestClass
{
    void Method()
    {
        if (true)
        {
            return;
        }
        else
        {
            return;
        }
    }
}
```
