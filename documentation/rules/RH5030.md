# RH5030 — Blank line after closing brace

| Property | Value |
|----------|-------|
| **ID** | RH5030 |
| **Category** | Layout |
| **Severity** | Warning |
| **Code Fix** | ✓ |

## Description

A blank line must appear after a closing brace (`}`) when a statement follows on the next line.

## Why is this a problem?

Without a blank line, a closing brace and the following statement run together visually, making it harder to spot where one logical block ends and the next independent statement begins. Adding the blank line makes the boundary obvious at a glance.

## How to fix it

Insert one blank line between the closing brace and the next statement. The code fix does this automatically.

## Examples

### Violation

```cs
internal void Execute(bool flag)
{
    if (flag)
    {
        Consume();
    }
    Consume(); // missing blank line after closing brace
}
```

### Correction

```cs
internal void Execute(bool flag)
{
    if (flag)
    {
        Consume();
    }

    Consume();
}
```
