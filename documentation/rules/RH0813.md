# RH0813 — Final array initializer items must not have trailing commas

| Property | Value |
|----------|-------|
| **ID** | RH0813 |
| **Category** | Formatting |
| **Severity** | Warning |
| **Code Fix** | ✓ |

## Description

This rule forbids a trailing comma after the final item in an array initializer.

## Why is this a problem?

A trailing comma on the last array item adds visual noise without making the initializer easier to read.

Keeping the final item free of a trailing comma makes array initializers consistent with the preferred formatting style enforced by this analyzer package.

## How to fix it

Remove the comma after the final array item. If the item has an attached comment, keep the comment with the item and remove only the comma.

The code fix and formatter can apply this automatically.

## Examples

### Violation

```cs
internal class Example
{
    private static readonly int[] Values = new int[]
    {
        1,
        2, // final value
    };
}
```

### Correction

```cs
internal class Example
{
    private static readonly int[] Values = new int[]
    {
        1,
        2 // final value
    };
}
```
