# RH0376 — Comments must be on their own line

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH0376       |
| **Category** | Formatting   |
| **Severity** | Warning      |
| **Code Fix** | ✓            |

## Description

This rule requires comments to appear on their own line. A comment must not share a line with code before it or after it.

## Why is this a problem?

Inline comments make statements and expressions harder to scan because code and commentary compete for the same space. Keeping comments on separate lines improves readability and makes intent clearer.

## How to fix it

Move the comment to a separate line near the code it describes. In most cases, place it directly above the related statement or expression.

## Examples

### Violation

```cs
if (a == b) // comment
if (a == b) /* comment */
if (a /* comment */ == b)
```

### Correction

```cs
// comment
if (a == b)

/* comment */
if (a == b)

/* comment */
if (a == b)
```
