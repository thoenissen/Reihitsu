# RH0206 — Interface names should use the I prefix and PascalCase

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH0206       |
| **Category** | Naming       |
| **Severity** | Warning      |
| **Code Fix** | ✓            |

## Description

Checks that interface names follow PascalCase naming conventions with the required `I` prefix. Each word in the name should start with an uppercase letter, with no underscores or other separators.

## Why is this a problem?

PascalCase with an `I` prefix for interface names is a fundamental C# naming convention. Inconsistent casing reduces readability and makes it harder for developers to distinguish interfaces from other types.

## How to fix it

Rename the interface to use PascalCase with an `I` prefix.

## Examples

### Violation

```cs
public interface myInterface
{
}
```

### Correction

```cs
public interface IMyInterface
{
}
```
