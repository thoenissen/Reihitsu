# RH0325 — Expression style methods should not be used

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH0325       |
| **Category** | Formatting   |
| **Severity** | Warning      |
| **Code Fix** | ✔️            |

## Description

This rule disallows expression-bodied methods (methods using `=>`). All methods should use block bodies with explicit `return` statements for consistency and clarity.

## Why is this a problem?

Expression-bodied methods can reduce readability for complex expressions and create inconsistency when some methods use block bodies and others use expression bodies. Requiring block bodies ensures a uniform coding style across the codebase.

## How to fix it

Replace the expression-bodied method with a block body containing a `return` statement (or just the statement for void methods).

## Examples

### Violation

```cs
public int GetValue() => 42;
```

### Correction

```cs
public int GetValue()
{
    return 42;
}
```
