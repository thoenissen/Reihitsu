# RH0409 - Private records must be documented

| Property | Value |
|----------|-------|
| **ID** | RH0409 |
| **Category** | Documentation |
| **Severity** | Warning |
| **Code Fix** | ❌ |

## Description

This rule requires private record declarations to have XML documentation. A declaration counts as documented when it has a &lt;summary&gt; tag or uses &lt;inheritdoc/&gt;.

## Why is this a problem?

Undocumented private record declarations make the code harder to understand because readers have to infer intent from implementation details instead of reading a clear contract.

## How to fix it

Add a meaningful XML comment above the declaration. In most cases a short &lt;summary&gt; is the best fix.

## Examples

### Violation

```cs
/// <summary>
/// Hosts nested models
/// </summary>
internal class Container
{
    private record Snapshot(int Count);
}
```

### Correction

```cs
/// <summary>
/// Hosts nested models
/// </summary>
internal class Container
{
    /// <summary>
    /// Captures the current item count for later comparison
    /// </summary>
    private record Snapshot(int Count);
}
```
