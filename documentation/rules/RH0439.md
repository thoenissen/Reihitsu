# RH0439 - Void return value must not be documented

| Property | Value |
|----------|-------|
| **ID** | RH0439 |
| **Category** | Documentation |
| **Severity** | Warning |
| **Code Fix** | ✓ |

## Description

This rule forbids `<returns>` tags on methods and delegates that return `void`.

## Why is this a problem?

A `void` member has no return value to describe. Keeping a `<returns>` tag in the documentation is misleading and suggests behavior that does not exist.

## How to fix it

Remove the `<returns>` tag from the documentation comment.

## Examples

### Violation

```cs
/// <summary>
/// Saves the customer.
/// </summary>
/// <returns>Nothing</returns>
internal void Save(Customer customer)
{
}
```

### Correction

```cs
/// <summary>
/// Saves the customer
/// </summary>
internal void Save(Customer customer)
{
}
```
