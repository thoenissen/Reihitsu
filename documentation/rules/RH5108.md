# RH5108 — Parameter list must follow declaration

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH5108       |
| **Category** | Layout |
| **Severity** | Warning      |
| **Code Fix** | ✓            |

## Description

The parameter list of a method or constructor must begin on the same line as the declaration, immediately following the opening parenthesis.

## Why is this a problem?

Starting the parameter list on the next line separates the declaration from its parameters and makes the signature harder to read at a glance.

## How to fix it

Move the first parameter to the same line as the opening parenthesis. The code fix applies this change automatically.

## Examples

### Violation

```cs
internal class TestClass
{
    void Method(
        int value)
    {
    }
}
```

### Correction

```cs
internal class TestClass
{
    void Method(int value)
    {
    }
}
```

## Known limitations

This rule currently inspects only method and constructor declarations. The formatter normalizes every parameter list (including local functions, operators, delegates, and records), so those other declaration shapes are not yet flagged by this analyzer.
