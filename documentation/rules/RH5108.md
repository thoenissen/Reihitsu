# RH5108 — Parameter list must follow declaration

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH5108       |
| **Category** | Layout |
| **Severity** | Warning      |
| **Code Fix** | ✓            |

## Description

The parameter list must begin on the same line as the declaration, immediately following the opening parenthesis. This applies to methods, constructors, local functions, operators, conversion operators, delegate declarations, and record (and record struct) primary constructors.

## Why is this a problem?

Starting the parameter list on the next line separates the declaration from its parameters and makes the signature harder to read at a glance.

## How to fix it

Move the first parameter to the same line as the opening parenthesis. The code fix applies this change automatically.

When a comment or directive sits in the gap between the opening parenthesis and the first parameter, the shape is not reported, because the formatter refuses to collapse across that trivia.

## Examples

### Violation

```cs
internal class TestClass
{
    void Method(
        int value)
    {
    }
}
```

### Correction

```cs
internal class TestClass
{
    void Method(int value)
    {
    }
}
```
