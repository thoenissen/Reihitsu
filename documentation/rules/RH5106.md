# RH5106 — Closing parenthesis must be on line of last argument

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH5106       |
| **Category** | Layout |
| **Severity** | Warning      |
| **Code Fix** | ✓            |

## Description

The closing parenthesis must appear on the same line as the last argument or parameter, not on a line of its own.

## Why is this a problem?

A closing parenthesis on its own line adds a visually empty line between the last argument and subsequent code, making parameter lists harder to scan and taking up unnecessary vertical space.

## How to fix it

Move the closing parenthesis to the end of the line that contains the last argument or parameter. The code fix applies this change automatically.

When a comment or directive sits in the gap before the closing parenthesis, the shape is not reported, because the formatter refuses to collapse across that trivia.

## Examples

### Violation

```cs
internal class TestClass
{
    void Method1(int first, int second
                )
    {
    }

    void Method2(int first,
                int second
                )
    {
    }
}
```

### Correction

```cs
internal class TestClass
{
    void Method1(int first, int second)
    {
    }

    void Method2(int first,
                 int second)
    {
    }
}
```

## Known limitations

This rule currently inspects only method and constructor declarations. The formatter normalizes every parameter list (including local functions, operators, delegates, and records), so those other declaration shapes are not yet flagged by this analyzer.
