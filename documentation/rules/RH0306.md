# RH0306 — do-Statement should be preceded by a blank line

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH0306       |
| **Category** | Formatting   |
| **Severity** | Warning      |
| **Code Fix** | ✗            |

## Description

This rule checks that `do` statements are preceded by a blank line. The rule does not apply when the `do` statement is the first statement in a block.

## Why is this a problem?

Separating statements with blank lines improves code readability by visually grouping related logic. A blank line before a `do` loop clearly marks the start of the loop construct and separates it from preceding code.

## How to fix it

Add a blank line before the `do` statement. The rule does not apply when the statement is the first statement in a block.

## Examples

### Violation

```cs
var x = 0;
do
{
    x++;
}
while (x < 10);
```

### Correction

```cs
var x = 0;

do
{
    x++;
}
while (x < 10);
```
