# RH5305 — Collection expressions should be formatted correctly

| Property | Value |
|----------|-------|
| **ID** | RH5305 |
| **Category** | Layout |
| **Severity** | Warning |
| **Code Fix** | ✓ |

## Description

This rule enforces consistent layout for C# collection expressions (`[...]`).

## Why is this a problem?

Collection expressions are often used in assignments, returns, and argument lists. If brackets and elements are inconsistently placed, the structure is harder to scan and nested code becomes difficult to read.

## How to fix it

Use one of these layouts:

1. Keep everything on one line.
2. Use a multiline layout where opening and closing brackets are aligned and each element is on its own line between them.

The code fix reformats the surrounding code automatically using the formatter.

## Examples

### Violation

```cs
int[] values = [
    1, 2,
    3
          ];
```

### Correction

```cs
int[] values = [
                   1,
                   2,
                   3
               ];
```
