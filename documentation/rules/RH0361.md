# RH0361 — Element must not be on a single line

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH0361       |
| **Category** | Formatting   |
| **Severity** | Warning      |
| **Code Fix** | ✓            |

## Description

A type declaration body must not be written on a single line. The opening brace, body, and closing brace must each appear on separate lines.

## Why is this a problem?

Collapsing a type declaration onto one line hides its structure and makes it harder to add members or read the declaration at a glance.

## How to fix it

Expand the declaration so the opening brace and closing brace are each on their own lines. The code fix applies this change automatically.

## Examples

### Violation

```cs
internal class TestClass { }
```

### Correction

```cs
internal class TestClass
{
}
```
