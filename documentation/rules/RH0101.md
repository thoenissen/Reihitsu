# RH0101 — Private auto-implemented properties should not be used

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH0101       |
| **Category** | Design       |
| **Severity** | Warning      |
| **Code Fix** | ✓            |

## Description

Private auto-implemented properties should not be used. This rule flags private properties with auto-implemented getters and setters that provide no additional logic.

## Why is this a problem?

Private auto-properties offer no benefit over plain fields and add unnecessary syntactic complexity. Fields are simpler, more direct, and clearly convey that the member is an internal implementation detail with no encapsulation logic.

## How to fix it

Replace the private auto-implemented property with a private field.

## Examples

### Violation

```cs
private int Count { get; set; }
```

### Correction

```cs
private int _count;
```
