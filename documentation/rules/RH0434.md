# RH0434 - Element parameter documentation must match element parameters

| Property | Value |
|----------|-------|
| **ID** | RH0434 |
| **Category** | Documentation |
| **Severity** | Warning |
| **Code Fix** | ❌ |

## Description

This rule requires `<param>` tags to match the declaration's parameters and appear in the same order.

## Why is this a problem?

Mismatched or reordered parameter documentation makes XML comments misleading. Readers can easily assign the wrong meaning to a parameter when the docs do not line up with the signature.

## How to fix it

Rename or reorder the `<param>` tags so they exactly match the parameter list.

## Examples

### Violation

```cs
/// <summary>
/// Saves the customer
/// </summary>
/// <param name="overwrite">Whether an existing customer may be replaced</param>
/// <param name="customer">The customer to save</param>
internal void Save(Customer customer, bool overwrite)
{
}
```

### Correction

```cs
/// <summary>
/// Saves the customer.
/// </summary>
/// <param name="customer">The customer to save</param>
/// <param name="overwrite">Whether an existing customer may be replaced</param>
internal void Save(Customer customer, bool overwrite)
{
}
```
