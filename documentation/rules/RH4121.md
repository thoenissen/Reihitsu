# RH4121 — Type parameter names should start with an uppercase 'T'

| Property | Value |
|----------|-------|
| **ID** | RH4121 |
| **Category** | Naming |
| **Severity** | Warning |
| **Code Fix** | ✓ |

## Description

Ensures generic type parameters on types, methods, delegates, and local functions are either a single uppercase `T` or an uppercase `T` followed by a PascalCase name, for example `TKey` or `TValue`.

## Why is this a problem?

A type parameter named `Key` or `Value` is indistinguishable from a regular type at the use site. The `T` prefix is the established .NET convention (`TKey`, `TValue`, `TResult`) and makes a name read unambiguously as a type parameter.

## How to fix it

Rename the type parameter so it starts with an uppercase `T` followed by a PascalCase name, or use a single `T` when the declaration has only one type parameter.

## Examples

### Violation

```cs
public class Cache<Key, Value>
{
    public Value Get(Key key)
    {
        return default;
    }
}
```

### Correction

```cs
public class Cache<TKey, TValue>
{
    public TValue Get(TKey key)
    {
        return default;
    }
}
```
