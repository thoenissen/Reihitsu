# RH0440 - Generic type parameters must be documented

| Property | Value |
|----------|-------|
| **ID** | RH0440 |
| **Category** | Documentation |
| **Severity** | Warning |
| **Code Fix** | ❌ |

## Description

This rule requires a `<typeparam>` tag for each generic type parameter on documented types, partial types, methods, and delegates.

## Why is this a problem?

Generic type parameters often carry important expectations about the caller's input. Without documentation, their purpose is easy to misunderstand.

## How to fix it

Add a `<typeparam>` tag for every generic type parameter, including partial declarations that carry the type documentation, and explain what it represents.

## Examples

### Violation

```cs
/// <summary>
/// Creates a repository instance
/// </summary>
internal class Repository<T>
{
}
```

### Correction

```cs
/// <summary>
/// Creates a repository instance
/// </summary>
/// <typeparam name="T">The entity type stored by the repository.</typeparam>
internal class Repository<T>
{
}
```
