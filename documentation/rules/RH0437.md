# RH0437 - Element return value must be documented

| Property | Value |
|----------|-------|
| **ID** | RH0437 |
| **Category** | Documentation |
| **Severity** | Warning |
| **Code Fix** | ❌ |

## Description

This rule requires a `<returns>` tag for documented methods and delegates that return a value.

## Why is this a problem?

Return values are part of the contract. If they are undocumented, callers have to infer what the method gives back and when.

## How to fix it

Add a `<returns>` tag that explains the returned value.

## Examples

### Violation

```cs
/// <summary>
/// Gets the customer count
/// </summary>
internal int GetCustomerCount()
{
    return 0;
}
```

### Correction

```cs
/// <summary>
/// Gets the customer count
/// </summary>
/// <returns>The number of customers currently loaded</returns>
internal int GetCustomerCount()
{
    return 0;
}
```
