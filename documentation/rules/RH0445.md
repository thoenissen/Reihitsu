# RH0445 - <inheritdoc> must be used with inheriting class

| Property | Value |
|----------|-------|
| **ID** | RH0445 |
| **Category** | Documentation |
| **Severity** | Warning |
| **Code Fix** | ❌ |

## Description

This rule only allows `<inheritdoc/>` on declarations that actually inherit documentation from a base member, base type, or implemented interface.

## Why is this a problem?

Using `<inheritdoc/>` on a standalone declaration suggests inherited behavior that does not exist. The resulting documentation is misleading and may leave the declaration effectively undocumented.

## How to fix it

Replace `<inheritdoc/>` with real documentation, or only use it on declarations that override, implement, or inherit from another declaration.

## Examples

### Violation

```cs
/// <inheritdoc/>
internal class CustomerRepository
{
}
```

### Correction

```cs
/// <summary>
/// Stores and retrieves customers
/// </summary>
internal class CustomerRepository
{
}
```
