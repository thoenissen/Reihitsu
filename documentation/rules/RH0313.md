# RH0313 — break statement should be followed by a blank line

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH0313       |
| **Category** | Formatting   |
| **Severity** | Warning      |
| **Code Fix** | ✔️            |

## Description

This rule checks that `break` statements in switch cases are followed by a blank line before the next case label. This ensures visual separation between switch cases for improved readability.

## Why is this a problem?

Without a blank line after a `break` statement, consecutive switch cases appear visually run together. This makes it harder to distinguish where one case ends and another begins, especially in switch blocks with many cases.

## How to fix it

Add a blank line after the `break` statement and before the next `case` or `default` label.

## Examples

### Violation

```cs
case 1:
    DoSomething();
    break;
case 2:
    DoSomethingElse();
    break;
```

### Correction

```cs
case 1:
    DoSomething();
    break;

case 2:
    DoSomethingElse();
    break;
```
