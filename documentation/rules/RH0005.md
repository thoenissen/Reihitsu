# RH0005 — Comments must contain text

| Property | Value |
|----------|-------|
| **ID** | RH0005 |
| **Category** | Clarity |
| **Severity** | Warning |
| **Code Fix** | ✓ |

## Description

This rule disallows empty comments such as `//` or `/* */` when they contain no useful content.

## Why is this a problem?

Empty comments add noise but provide no information. They make code look unfinished and distract from comments that actually matter.

## How to fix it

Remove the empty comment or replace it with meaningful text.

## Examples

### Violation

```cs
public void Run()
{
    //
    DoWork();
}
```

### Correction

```cs
public void Run()
{
    DoWork();
}
```
