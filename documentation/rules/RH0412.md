# RH0412 - Non-private enums must be documented

| Property | Value |
|----------|-------|
| **ID** | RH0412 |
| **Category** | Documentation |
| **Severity** | Warning |
| **Code Fix** | ❌ |

## Description

This rule requires non-private enum declarations to have XML documentation. A declaration counts as documented when it has a &lt;summary&gt; tag or uses &lt;inheritdoc/&gt;.

## Why is this a problem?

Undocumented non-private enum declarations make the code harder to understand because readers have to infer intent from implementation details instead of reading a clear contract.

## How to fix it

Add a meaningful XML comment above the declaration. In most cases a short &lt;summary&gt; is the best fix.

## Examples

### Violation

```cs
internal enum OrderState
{
    Open
}
```

### Correction

```cs
/// <summary>
/// Represents the processing state of an order
/// </summary>
internal enum OrderState
{
    Open
}
```
