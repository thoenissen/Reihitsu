# RH0331 — The first argument should be on the same line as the opening parenthesis

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH0331       |
| **Category** | Formatting   |
| **Severity** | Warning      |
| **Code Fix** | ✓            |

## Description

This rule checks that the first argument of a method or constructor call is on the same line as the opening parenthesis. When arguments are placed on the next line after the opening parenthesis, it makes alignment inconsistent and wastes vertical space.

## Why is this a problem?

Placing the first argument on a new line after the opening parenthesis creates unnecessary vertical distance between the method name and its first argument, reducing readability.

## How to fix it

Move the first argument to the same line as the opening parenthesis. The code fix will automatically collapse the line break between the opening parenthesis and the first argument.

## Examples

### Violation

```cs
a.Call(
    "test1",
    "test2");
```

### Correction

```cs
a.Call("test1",
       "test2");
```
