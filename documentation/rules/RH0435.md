# RH0435 - Element parameter documentation must declare parameter name

| Property | Value |
|----------|-------|
| **ID** | RH0435 |
| **Category** | Documentation |
| **Severity** | Warning |
| **Code Fix** | ❌ |

## Description

This rule requires each `<param>` tag to declare the parameter name through its `name` attribute.

## Why is this a problem?

Without the parameter name, the comment cannot be tied to a specific argument. The documentation becomes ambiguous and tooling cannot map it to the signature.

## How to fix it

Add the correct `name` attribute to every `<param>` tag.

## Examples

### Violation

```cs
/// <summary>
/// Saves the customer
/// </summary>
/// <param>The customer to save</param>
internal void Save(Customer customer)
{
}
```

### Correction

```cs
/// <summary>
/// Saves the customer
/// </summary>
/// <param name="customer">The customer to save</param>
internal void Save(Customer customer)
{
}
```
