# RH0419 - Private constructors must be documented

| Property | Value |
|----------|-------|
| **ID** | RH0419 |
| **Category** | Documentation |
| **Severity** | Warning |
| **Code Fix** | ❌ |

## Description

This rule requires private constructor declarations to have XML documentation. A declaration counts as documented when it has a &lt;summary&gt; tag or uses &lt;inheritdoc/&gt;.

## Why is this a problem?

Undocumented private constructor declarations make the code harder to understand because readers have to infer intent from implementation details instead of reading a clear contract.

## How to fix it

Add a meaningful XML comment above the declaration. In most cases a short &lt;summary&gt; is the best fix.

## Examples

### Violation

```cs
/// <summary>
/// Represents a singleton
/// </summary>
internal class Singleton
{
    private Singleton()
    {
    }
}
```

### Correction

```cs
/// <summary>
/// Represents a singleton
/// </summary>
internal class Singleton
{
    /// <summary>
    /// Prevents external construction of the singleton instance
    /// </summary>
    private Singleton()
    {
    }
}
```
