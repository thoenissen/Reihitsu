# RH0455 - Extension declaration type parameters must be documented

| Property | Value |
|----------|-------|
| **ID** | RH0455 |
| **Category** | Documentation |
| **Severity** | Warning |
| **Code Fix** | ❌ |

## Description

This rule requires each generic type parameter declared on an `extension<T>(...)` declaration to have matching `<typeparam>` documentation.

## Why is this a problem?

Generic type parameters are part of the extension declaration contract. Without `<typeparam>` documentation, callers have less context about the role of each generic type argument.

## How to fix it

Add `<typeparam>` tags for all type parameters declared by the extension declaration and keep the names aligned.

## Examples

### Violation

```cs
using System.Collections.Generic;

public static class Extensions
{
    /// <summary>Provides collection helpers.</summary>
    extension<T>(IEnumerable<T> values)
    {
    }
}
```

### Correction

```cs
using System.Collections.Generic;

public static class Extensions
{
    /// <summary>Provides collection helpers.</summary>
    /// <typeparam name="T">The element type.</typeparam>
    extension<T>(IEnumerable<T> values)
    {
    }
}
```
