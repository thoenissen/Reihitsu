# RH0368 — Chained statement blocks must not be preceded by blank line

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH0368       |
| **Category** | Formatting   |
| **Severity** | Warning      |
| **Code Fix** | ✓            |

## Description

An `else`, `catch`, or `finally` keyword must appear on the line immediately following the closing brace of the preceding block, without a blank line in between.

## Why is this a problem?

A blank line between chained blocks visually breaks the connection between `if`/`else` or `try`/`catch`/`finally` clauses, making the overall control flow harder to follow.

## How to fix it

Remove the blank line between the closing brace and the chained keyword. The code fix applies this change automatically.

## Examples

### Violation

```cs
internal class TestClass
{
    void Method()
    {
        if (true)
        {
        }

        else
        {
        }
    }
}
```

### Correction

```cs
internal class TestClass
{
    void Method()
    {
        if (true)
        {
        }
        else
        {
        }
    }
}
```
