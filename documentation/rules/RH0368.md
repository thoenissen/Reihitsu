# RH0368 — Chained statement blocks must not be preceded by blank line

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH0368       |
| **Category** | Formatting   |
| **Severity** | Warning      |
| **Code Fix** | ✓            |

## Description

This rule ensures that chained statement blocks must not be preceded by blank line.

## Why is this a problem?

Inconsistent formatting makes code harder to read and creates unnecessary noise in diffs.

## How to fix it

Adjust the affected formatting manually or apply the provided code fix.

## Examples

### Violation

```cs
internal class TestClass
{
    void Method()
    {
        if (true)
        {
        }

        else
        {
        }
    }
}
```

### Correction

```cs
internal class TestClass
{
    void Method()
    {
        if (true)
        {
        }
        else
        {
        }
    }
}
```
