# RH0332 — Arguments should either all be on one line or each on its own line

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH0332       |
| **Category** | Formatting   |
| **Severity** | Warning      |
| **Code Fix** | ✓            |

## Description

This rule checks that arguments in a method or constructor call are either all on a single line or each on its own line. Mixing these styles — where some arguments share a line while others are on separate lines — is not permitted.

Multi-line arguments (e.g., lambda expressions or method chains passed as arguments) are allowed as long as each argument starts on its own line.

## Why is this a problem?

Inconsistent argument placement makes it harder to scan and understand method calls. A uniform style — either compact single-line or expanded one-per-line — improves readability.

## How to fix it

Either place all arguments on a single line, or place each argument on its own line. The code fix will automatically place each argument on its own line.

## Examples

### Violation

```cs
a.Call("test1", "test2",
       "test3");
```

### Correction

```cs
// All on one line
a.Call("test1", "test2", "test3");

// Each on its own line
a.Call("test1",
       "test2",
       "test3");

// Multi-line arguments are also valid when each starts on its own line
a.Call("test1",
       b.Call()
         .Chain(),
       "test3");
```
