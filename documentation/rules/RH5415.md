# RH5415 — Empty records should use semicolon declarations

| Property | Value |
|----------|-------|
| **ID** | RH5415 |
| **Category** | Layout |
| **Severity** | Warning |
| **Code Fix** | ✓ |

## Description

This rule prefers semicolon declarations for empty records.

Brace-based empty record bodies add noise when the declaration contains no members and the language version supports semicolon syntax.

## Why is this a problem?

An empty record often serves as a concise data marker or inheritance shell. Extra braces make that declaration heavier than necessary.

Using a semicolon declaration keeps the intent obvious while reducing structural clutter.

## How to fix it

Replace the empty brace body with a trailing semicolon.

The code fix and formatter only apply this when the body is truly empty. Comments, directives, and other meaningful trivia inside the braces are preserved by leaving the declaration unchanged.

## Examples

### Violation

```cs
internal record Example
{
}
```

### Correction

```cs
internal record Example;
```
