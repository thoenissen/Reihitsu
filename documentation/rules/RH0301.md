# RH0301 — #region and #endregion descriptions should match

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH0301       |
| **Category** | Formatting   |
| **Severity** | Warning      |
| **Code Fix** | ✓            |

## Description

This rule checks that the description text on a `#region` directive matches the description text on its corresponding `#endregion` directive. Mismatched descriptions can confuse developers about which region they are closing.

## Why is this a problem?

When `#region` and `#endregion` descriptions do not match, it becomes unclear which region is being closed. This can lead to confusion when navigating large files, especially when regions are nested or span many lines.

## How to fix it

Update the `#endregion` description to match the `#region` description exactly. The code fix will automatically correct the `#endregion` comment.

## Examples

### Violation

```cs
#region Methods

public void DoWork()
{
}

#endregion // Fields
```

### Correction

```cs
#region Methods

public void DoWork()
{
}

#endregion // Methods
```
