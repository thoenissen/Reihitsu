# RH5024 — Closing brace must not be preceded by blank line

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH5024       |
| **Category** | Layout |
| **Severity** | Warning      |
| **Code Fix** | ✓            |

## Description

The last line of content inside a block must not be followed by a blank line before the closing brace.

## Why is this a problem?

A blank line immediately before a closing brace adds vertical whitespace with no semantic value and makes the block feel artificially padded.

## How to fix it

Remove the blank line that precedes the closing brace. The code fix applies this change automatically.

## Examples

### Violation

```cs
internal class TestClass
{
    void Method()
    {
        int value = 0;

    }
}
```

### Correction

```cs
internal class TestClass
{
    void Method()
    {
        int value = 0;
    }
}
```

### Violation (blank line between an `#endregion` directive and the closing brace)

```cs
internal class TestClass
{
    #region Properties
    public int Value { get; set; }
    #endregion

}
```

### Correction

```cs
internal class TestClass
{
    #region Properties
    public int Value { get; set; }
    #endregion
}
```

> A blank line that precedes a trailing `#endregion` directive which already hugs the closing brace is allowed, so `#endregion` keeps its own blank-line padding inside the region.
