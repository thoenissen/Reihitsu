# RH3204 — Interpolated strings without interpolation should not use the dollar marker

| Property | Value |
|----------|-------|
| **ID** | RH3204 |
| **Category** | Clarity |
| **Severity** | Warning |
| **Code Fix** | ✓ |

## Description

This rule forbids the use of the interpolation marker (`$`) on string literals that do not contain any interpolation holes.

An interpolated string without holes still converts to `System.FormattableString` or `System.IFormattable` when the target type requires it, whereas a plain string literal does not. Removing the marker in those cases would either fail to compile (CS0029) or silently change overload resolution. The rule is therefore only reported when the string's converted type is `string`; interpolated strings converted to `FormattableString` or `IFormattable` are left untouched.

## Why is this a problem?

Using the interpolation marker on a string without interpolation adds noise and can mislead readers into expecting runtime interpolation. It also increases code inconsistency and violates clean code principles by including unused syntax.

Removing the unnecessary marker makes the intent clear: the string is a literal, not an interpolated expression.

## How to fix it

Remove the leading `$` (and any additional `$` markers for raw strings) from the interpolated string literal. The string contents remain unchanged, as they already represent a literal.

Standard and verbatim interpolated strings escape literal braces by doubling them (`{{` and `}}`). Because the resulting literal is no longer interpolated, these doubled braces are collapsed back to single braces so the runtime value is preserved (`$"a {{b}}"` becomes `"a {b}"`). Raw string literals do not use brace doubling, so their braces are left unchanged.

The code fix can apply this automatically. Marker removal depends on the converted type, so it is driven by the analyzer's semantic check rather than by the syntax-only formatter.

## Examples

### Violation

```cs
var message = $"System ready";
var path = $@"C:\Users\John";
var braces = $"a {{b}}";
var text = $$"""
           multiline
           raw string
           """;
```

### Correction

```cs
var message = "System ready";
var path = @"C:\Users\John";
var braces = "a {b}";
var text = """
           multiline
           raw string
           """;
```
