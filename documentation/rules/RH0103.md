# RH0103 — NotImplementedException should not be used

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH0103       |
| **Category** | Design       |
| **Severity** | Warning      |
| **Code Fix** | ✗            |

## Description

`NotImplementedException` should not be used. This rule flags any usage of `throw new NotImplementedException()` in the codebase.

## Why is this a problem?

The presence of `NotImplementedException` indicates incomplete implementation that was left as a placeholder. Shipping code with these exceptions leads to unexpected runtime failures and signals that the code is not production-ready.

## How to fix it

Implement the method body with the actual logic, or throw a more appropriate exception such as `NotSupportedException` if the operation is intentionally unsupported.

## Examples

### Violation

```cs
public void Process()
{
    throw new NotImplementedException();
}
```

### Correction

```cs
public void Process()
{
    // actual implementation
}
```
