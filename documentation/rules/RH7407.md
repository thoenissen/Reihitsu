# RH7407 — Non-override events should not be placed in override regions

| Property | Value |
|----------|-------|
| **ID** | RH7407 |
| **Category** | Organization |
| **Severity** | Warning |
| **Code Fix** | ❌ |

## Description

This rule forbids non-override events from being placed inside a top-level override region whose label is used by override events in the same type.

## Why is this a problem?

Override regions are meant to isolate inherited members that come from a particular base type. Placing ordinary events inside those regions blurs that inheritance boundary and makes override-focused scanning less dependable.

## How to fix it

Move each non-override event out of the override region and into a normal organizational region such as `Events`, or leave it ungrouped if that matches the surrounding style.

## Examples

### Violation

```cs
internal abstract class BaseProcessor
{
    public abstract event System.EventHandler Completed;
}

internal sealed class ReportProcessor : BaseProcessor
{
    #region BaseProcessor

    public override event System.EventHandler Completed
    {
        add
        {
        }
        remove
        {
        }
    }

    public event System.EventHandler Saved;

    #endregion // BaseProcessor
}
```

### Correction

```cs
internal abstract class BaseProcessor
{
    public abstract event System.EventHandler Completed;
}

internal sealed class ReportProcessor : BaseProcessor
{
    #region BaseProcessor

    public override event System.EventHandler Completed
    {
        add
        {
        }
        remove
        {
        }
    }

    #endregion // BaseProcessor

    #region Events

    public event System.EventHandler Saved;

    #endregion // Events
}
```
