# RH0013 — Do not use query syntax

| Property | Value |
|----------|-------|
| **ID** | RH0013 |
| **Category** | Clarity |
| **Severity** | Warning |
| **Code Fix** | ❌ |

## Description

This rule disallows LINQ query syntax and requires method syntax instead.

## Why is this a problem?

Method syntax is more consistent with the underlying LINQ API, composes more naturally with extension methods, and keeps the style uniform across the codebase.

## How to fix it

Rewrite the query as a method chain using operators such as `Where`, `Select`, and `OrderBy`.

## Examples

### Violation

```cs
var names = from person in people
            where person.IsActive
            select person.Name;
```

### Correction

```cs
var names = people.Where(person => person.IsActive)
                  .Select(person => person.Name);
```
