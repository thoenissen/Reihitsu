# RH0304 — if-Statement should be preceded by a blank line

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH0304       |
| **Category** | Formatting   |
| **Severity** | Warning      |
| **Code Fix** | ✔️            |

## Description

This rule checks that `if` statements are preceded by a blank line. The rule does not apply when the `if` statement is the first statement in a block.

## Why is this a problem?

Separating statements with blank lines improves code readability by visually grouping related logic. A blank line before an `if` statement clearly delineates the start of a conditional block from the preceding code.

## How to fix it

Add a blank line before the `if` statement. The rule does not apply when the statement is the first statement in a block.

## Examples

### Violation

```cs
var x = 1;
if (x > 0)
{
    Console.WriteLine("Positive");
}
```

### Correction

```cs
var x = 1;

if (x > 0)
{
    Console.WriteLine("Positive");
}
```
