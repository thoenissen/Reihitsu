# RH0399 — Collection initializer should be formatted correctly

| Property | Value |
|----------|-------|
| **ID** | RH0399 |
| **Category** | Formatting |
| **Severity** | Warning |
| **Code Fix** | ✓ |

## Description

This rule requires collection initializer braces and entries to be formatted under the `new` anchor.

## Why is this a problem?

Misaligned collection initializers make it harder to see where the creation expression starts and where each item belongs. Consistent brace placement keeps collection setup readable, especially in longer initializers.

## How to fix it

Move the opening brace onto its own line under `new`, put each entry on its own indented line, and align the closing brace with `new`. The code fix will reformat the initializer automatically.

## Examples

### Violation

```cs
var values = new List<int>
{
1,
    2
};
```

### Correction

```cs
var values = new List<int>
             {
                 1,
                 2
             };
```
