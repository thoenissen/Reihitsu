# RH0423 - Non-private properties must be documented

| Property | Value |
|----------|-------|
| **ID** | RH0423 |
| **Category** | Documentation |
| **Severity** | Warning |
| **Code Fix** | ❌ |

## Description

This rule requires non-private property declarations to have XML documentation. A declaration counts as documented when it has a &lt;summary&gt; tag or uses &lt;inheritdoc/&gt;.

## Why is this a problem?

Undocumented non-private property declarations make the code harder to understand because readers have to infer intent from implementation details instead of reading a clear contract.

## How to fix it

Add a meaningful XML comment above the declaration. In most cases a short &lt;summary&gt; is the best fix.

## Examples

### Violation

```cs
/// <summary>
/// Stores application settings
/// </summary>
internal class Settings
{
    internal string Name { get; }
}
```

### Correction

```cs
/// <summary>
/// Stores application settings
/// </summary>
internal class Settings
{
    /// <summary>
    /// Gets the display name of the current settings profile
    /// </summary>
    internal string Name { get; }
}
```
