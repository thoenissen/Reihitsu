# RH0216 — Const field names should be in PascalCase

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH0216       |
| **Category** | Naming       |
| **Severity** | Warning      |
| **Code Fix** | ✓            |

## Description

Checks that const field names follow PascalCase naming conventions regardless of their access modifier. Each word in the name should start with an uppercase letter, with no underscores or other separators.

## Why is this a problem?

PascalCase for constants is the standard C# naming convention. Using other conventions such as SCREAMING_CASE or snake_case is inconsistent with the .NET ecosystem and reduces readability.

## How to fix it

Rename the const field to use PascalCase.

## Examples

### Violation

```cs
private const int max_value = 100;
```

### Correction

```cs
private const int MaxValue = 100;
```
