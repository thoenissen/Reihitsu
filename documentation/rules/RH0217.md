# RH0217 — Private property names should be in PascalCase

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH0217       |
| **Category** | Naming       |
| **Severity** | Warning      |
| **Code Fix** | ✓            |

## Description

Checks that private property names follow PascalCase naming conventions. Each word in the name should start with an uppercase letter, with no underscores or other separators.

## Why is this a problem?

PascalCase for properties is a fundamental C# naming convention that applies regardless of access modifier. Inconsistent naming reduces readability and breaks expectations within the codebase.

## How to fix it

Rename the private property to use PascalCase.

## Examples

### Violation

```cs
private int count { get; set; }
```

### Correction

```cs
private int Count { get; set; }
```
