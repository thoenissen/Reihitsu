# RH0230 — Record struct names should be in PascalCase

| Property | Value |
|----------|-------|
| **ID** | RH0230 |
| **Category** | Naming |
| **Severity** | Warning |
| **Code Fix** | ✓ |

## Description

Ensures `record struct` type names use PascalCase.

## Why is this a problem?

Record structs are types and should follow the same naming style as other types. Inconsistent casing reduces readability and makes declarations look out of place.

## How to fix it

Rename the record struct so each word starts with an uppercase letter.

## Examples

### Violation

```cs
public record struct currencyvalue(decimal Amount);
```

### Correction

```cs
public record struct Currencyvalue(decimal Amount);
```
