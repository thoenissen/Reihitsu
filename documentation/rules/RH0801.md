# RH0801 — Switch expression braces should be anchored

| Property | Value |
|----------|-------|
| **ID** | RH0801 |
| **Category** | Formatting |
| **Severity** | Warning |
| **Code Fix** | ✓ |

## Description

This rule requires multi-line switch expressions to align their braces and arms to the switch-expression anchor.

## Why is this a problem?

Misaligned switch-expression braces make it harder to see where the expression starts and ends. Consistent arm indentation also makes each case easier to scan.

## How to fix it

Align the opening brace, closing brace, and each arm under the start of the governing expression. The code fix will reformat the enclosing scope automatically.

## Examples

### Violation

```cs
var state = status switch
    {
      0 => "Idle",
        1 => "Running",
        _ => "Unknown"
    };
```

### Correction

```cs
var state = status switch
            {
                0 => "Idle",
                1 => "Running",
                _ => "Unknown"
            };
```
