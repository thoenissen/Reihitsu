# RH8301 — Documentation lines must begin with single space

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH8301       |
| **Category** | Documentation |
| **Severity** | Warning      |
| **Code Fix** | ✓            |

## Description

Each `///` documentation comment line must have exactly one space between the triple-slash marker and the comment content.

## Why is this a problem?

Missing or extra spaces in documentation lines make doc comments visually inconsistent and harder to read in both source and generated documentation.

## How to fix it

Insert exactly one space after the `///` prefix. The code fix applies this change automatically.

## Examples

### Violation

```cs
internal class TestClass
{
    ///Summary
    void Method()
    {
    }
}
```

### Correction

```cs
internal class TestClass
{
    /// Summary
    void Method()
    {
    }
}
```
