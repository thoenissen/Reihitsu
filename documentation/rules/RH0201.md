# RH0201 — The name of the type should match the filename

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH0201       |
| **Category** | Naming       |
| **Severity** | Warning      |
| **Code Fix** | ✓            |

## Description

Checks that the first type declared in a file has a name that matches the file name. Supports `.razor.cs` code-behind files by matching the type name against the file name without the `.razor` suffix when a corresponding `.razor` file exists.

## Why is this a problem?

When a type name does not match its file name, developers cannot locate the type by browsing the file system. This breaks common conventions and makes navigation in large codebases significantly harder.

## How to fix it

Rename the file to match the type name, or rename the type to match the file name. The code fix renames the file.

## Examples

### Violation

```cs
// File: Foo.cs
public class Bar
{
}
```

### Correction

```cs
// File: Bar.cs (renamed to match the type)
public class Bar
{
}
```
