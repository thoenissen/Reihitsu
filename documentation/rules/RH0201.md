# RH0201 — The name of the type should match the filename

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH0201       |
| **Category** | Naming       |
| **Severity** | Warning      |
| **Code Fix** | ✓            |

## Description

Checks that the first type declared in a file has a name that matches the file name segment before the first dot. This means `View.xaml.cs`, `Component.razor.cs`, and split files such as `Type.Part1.cs` are all validated against `View`, `Component`, and `Type`.

## Why is this a problem?

When a type name does not match its file name, developers cannot locate the type by browsing the file system. This breaks common conventions and makes navigation in large codebases significantly harder.

## How to fix it

Rename the file to match the type name before the first dot, or rename the type to match the file name. The code fix renames the file and preserves any suffix after the first dot.

## Examples

### Violation

```cs
// File: Foo.cs
public class Bar
{
}
```

### Correction

```cs
// File: Bar.cs (renamed to match the type)
public class Bar
{
}
```

### Allowed dotted file names

```cs
// File: Component.Part1.cs
public partial class Component
{
}
```
