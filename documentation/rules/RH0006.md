# RH0006 — Use `string.Empty` for empty strings

| Property | Value |
|----------|-------|
| **ID** | RH0006 |
| **Category** | Clarity |
| **Severity** | Warning |
| **Code Fix** | ✓ |

## Description

This rule prefers `string.Empty` over an empty string literal.

## Why is this a problem?

`string.Empty` makes the intent explicit and avoids a value that is easy to miss when scanning code.

## How to fix it

Replace `""` with `string.Empty` where a compile-time constant is not required.

## Examples

### Violation

```cs
var name = "";
```

### Correction

```cs
var name = string.Empty;
```
