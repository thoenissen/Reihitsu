# RH0446 - Do not use placeholder elements

| Property | Value |
|----------|-------|
| **ID** | RH0446 |
| **Category** | Documentation |
| **Severity** | Warning |
| **Code Fix** | ✓ |

## Description

This rule forbids `<placeholder>` elements in XML documentation.

## Why is this a problem?

Placeholder elements mark unfinished documentation. Leaving them in committed code makes the documentation look incomplete and distracts readers from the real contract.

## How to fix it

Remove the `<placeholder>` tags and keep only the final documentation text.

## Examples

### Violation

```cs
/// <summary>
/// Saves the <placeholder>current customer</placeholder>
/// </summary>
internal void Save()
{
}
```

### Correction

```cs
/// <summary>
/// Saves the current customer
/// </summary>
internal void Save()
{
}
```
