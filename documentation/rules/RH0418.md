# RH0418 - Non-private constructors must be documented

| Property | Value |
|----------|-------|
| **ID** | RH0418 |
| **Category** | Documentation |
| **Severity** | Warning |
| **Code Fix** | ❌ |

## Description

This rule requires non-private constructor declarations to have XML documentation. A declaration counts as documented when it has a &lt;summary&gt; tag or uses &lt;inheritdoc/&gt;.

## Why is this a problem?

Undocumented non-private constructor declarations make the code harder to understand because readers have to infer intent from implementation details instead of reading a clear contract.

## How to fix it

Add a meaningful XML comment above the declaration. In most cases a short &lt;summary&gt; is the best fix.

## Examples

### Violation

```cs
/// <summary>
/// Represents a formatter
/// </summary>
internal class Formatter
{
    internal Formatter()
    {
    }
}
```

### Correction

```cs
/// <summary>
/// Represents a formatter
/// </summary>
internal class Formatter
{
    /// <summary>
    /// Initializes a formatter with the default configuration
    /// </summary>
    internal Formatter()
    {
    }
}
```
