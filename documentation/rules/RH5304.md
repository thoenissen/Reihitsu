# RH5304 — Nested collection initializer assignments should be formatted correctly

| Property | Value |
|----------|-------|
| **ID** | RH5304 |
| **Category** | Layout |
| **Severity** | Warning |
| **Code Fix** | ✓ |

## Description

This rule requires member assignments with nested collection initializers within object initializers to follow one of two valid layouts: either the entire nested collection initializer stays on one line, or the collection uses a multiline block with aligned braces.

## Why is this a problem?

Inconsistent placement of braces and poor indentation alignment in nested collection initializers makes code harder to scan and understand. When the opening and closing braces are not aligned, or the brace position is unpredictable, it becomes difficult to visually track the structure of the initializer.

## How to fix it

Ensure that:
1. The assignment target and `=` token are on the same line
2. The opening `{` is on the same line as the assignment (`=`)
3. Single-line nested collections keep all elements on that same line
4. Multiline nested collections align opening `{` and closing `}` in the same column and keep each element on its own line

The code fix will reformat the nested collection initializer automatically using the formatter.

## Examples

### Violation — Misaligned braces

```cs
var value = new Example
            {
                Values = {
                        1,
                        2
                    }
            };

var value = new Example
            {
                Values =
                   {
                        1,
                        2
                   }
            };
```

### Correction

```cs
// Single line
var value = new Example
            {
                Values = { 1, 2 }
            };

// Multi line
var value = new Example
            {
                Values = {
                             1,
                             2
                         }
            };
```
