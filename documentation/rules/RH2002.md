# RH2002 — Async void methods should not be used

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH2002       |
| **Category** | Design |
| **Severity** | Warning      |
| **Code Fix** | ✗            |

## Description

Async void methods should not be used. This rule flags `async void` shapes declared as `async void` instead of `async Task` or `async Task<T>`. The following shapes are flagged:

- Ordinary methods declared `async void`.
- Local functions declared `async void`.
- Anonymous functions (lambdas and anonymous methods) that are `async` and bound to a `void`-returning delegate (for example `async () => ...` assigned to an `Action`).

## Why is this a problem?

Async void methods cannot be awaited, which means callers have no way to know when the operation completes. Exceptions thrown inside async void methods cannot be caught by the caller and will crash the process. They are also difficult to unit test reliably. The same hazards apply to `async void` local functions and to `async` lambdas/anonymous methods bound to `void`-returning delegates.

## How to fix it

Change the return type from `void` to `Task` or `Task<T>`.

## Examples

### Violation

```cs
private async void LoadDataAsync()
{
    await Task.Delay(1000);
}
```

### Correction

```cs
private async Task LoadDataAsync()
{
    await Task.Delay(1000);
}
```
