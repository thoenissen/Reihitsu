# RH5112 — Wrapped fluent calls should keep the first call on the original line

| Property | Value |
|----------|-------|
| **ID** | RH5112 |
| **Category** | Layout |
| **Severity** | Warning |
| **Code Fix** | ✓ |

## Description

This rule requires wrapped fluent calls to keep the first call on the original line. When later calls still need to wrap, they may continue on following lines.

## Why is this a problem?

Breaking before the first fluent call makes the expression harder to scan because the starting point is separated from the object being configured or queried. Keeping the first call on the original line makes the expression easier to follow while still allowing longer call sequences to wrap clearly.

## How to fix it

Move the first wrapped fluent call back onto the original line. Leave later calls on separate lines when the expression still needs to wrap.

If there is a comment directly above the first wrapped call, keep that call layout as written.

## Examples

### Violation

```cs
var pipeline = new PipelineBuilder()
    .UseLogging()
    .UseValidation()
    .UseExecution();
```

### Correction

```cs
var pipeline = new PipelineBuilder().UseLogging()
                                    .UseValidation()
                                    .UseExecution();
```
