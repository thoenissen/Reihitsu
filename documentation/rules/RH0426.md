# RH0426 - Private indexers must be documented

| Property | Value |
|----------|-------|
| **ID** | RH0426 |
| **Category** | Documentation |
| **Severity** | Warning |
| **Code Fix** | ❌ |

## Description

This rule requires private indexer declarations to have XML documentation. A declaration counts as documented when it has a &lt;summary&gt; tag or uses &lt;inheritdoc/&gt;.

## Why is this a problem?

Undocumented private indexer declarations make the code harder to understand because readers have to infer intent from implementation details instead of reading a clear contract.

## How to fix it

Add a meaningful XML comment above the declaration. In most cases a short &lt;summary&gt; is the best fix.

## Examples

### Violation

```cs
/// <summary>
/// Stores values by position
/// </summary>
internal class Buffer
{
    private int this[int index] => index;
}
```

### Correction

```cs
/// <summary>
/// Stores values by position
/// </summary>
internal class Buffer
{
    /// <summary>
    /// Gets the internally cached value at the specified index
    /// </summary>
    private int this[int index] => index;
}
```
