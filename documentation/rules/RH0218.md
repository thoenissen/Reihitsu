# RH0218 — Protected property names should be in PascalCase

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH0218       |
| **Category** | Naming       |
| **Severity** | Warning      |
| **Code Fix** | ✓            |

## Description

Checks that protected property names follow PascalCase naming conventions. Each word in the name should start with an uppercase letter, with no underscores or other separators.

## Why is this a problem?

PascalCase for properties is a fundamental C# naming convention that applies regardless of access modifier. Inconsistent naming reduces readability and breaks expectations for derived classes.

## How to fix it

Rename the protected property to use PascalCase.

## Examples

### Violation

```cs
protected int count { get; set; }
```

### Correction

```cs
protected int Count { get; set; }
```
