# RH0334 — C# source files should be encoded as UTF-8 with BOM

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH0334       |
| **Category** | Formatting   |
| **Severity** | Warning      |
| **Code Fix** | ✓            |

## Description

This rule checks whether `.cs` files are encoded as UTF-8 with a byte order mark (BOM). Using a consistent file encoding avoids accidental encoding drift and keeps source files aligned with the repository convention.

## Why is this a problem?

Different encodings in the same codebase can lead to inconsistent diffs, editor behavior, and build-time surprises. Enforcing UTF-8 with BOM keeps the representation of C# source files predictable.

## How to fix it

Re-save the file as UTF-8 with BOM. The code fix updates the document encoding without changing the source text.
