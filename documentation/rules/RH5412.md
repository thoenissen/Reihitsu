# RH5412 — Empty classes should use semicolon declarations

| Property | Value |
|----------|-------|
| **ID** | RH5412 |
| **Category** | Layout |
| **Severity** | Warning |
| **Code Fix** | ✓ |

## Description

This rule prefers semicolon declarations for empty classes.

Brace-based empty class bodies add noise when the declaration contains no members and the language version supports semicolon syntax.

## Why is this a problem?

An empty class body communicates no implementation detail, yet it still adds two braces and an extra block shape to scan.

Using a semicolon declaration keeps empty marker types, placeholders, and simple inheritance declarations concise without changing semantics.

## How to fix it

Replace the empty brace body with a trailing semicolon.

The code fix and formatter only apply this when the body is truly empty. Comments, directives, and other meaningful trivia around the braces — including a comment between the header and the open brace — are preserved by leaving the declaration unchanged.

## Examples

### Violation

```cs
internal class Example
{
}
```

### Correction

```cs
internal class Example;
```
