# RH0379 — Comma must be on same line as previous parameter

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH0379       |
| **Category** | Formatting   |
| **Severity** | Warning      |
| **Code Fix** | ✓            |

## Description

This rule ensures that comma must be on same line as previous parameter.

## Why is this a problem?

Inconsistent formatting makes code harder to read and creates unnecessary noise in diffs.

## How to fix it

Adjust the affected formatting manually or apply the provided code fix.

## Examples

### Violation

```cs
internal class TestClass
{
    void Method(int first
               ,int second)
    {
    }
}
```

### Correction

```cs
internal class TestClass
{
    void Method(int first,
                int second)
    {
    }
}
```
