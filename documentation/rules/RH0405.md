# RH0405 - Private structs must be documented

| Property | Value |
|----------|-------|
| **ID** | RH0405 |
| **Category** | Documentation |
| **Severity** | Warning |
| **Code Fix** | ❌ |

## Description

This rule requires private struct declarations to have XML documentation. A declaration counts as documented when it has a &lt;summary&gt; tag or uses &lt;inheritdoc/&gt;.

## Why is this a problem?

Undocumented private struct declarations make the code harder to understand because readers have to infer intent from implementation details instead of reading a clear contract.

## How to fix it

Add a meaningful XML comment above the declaration. In most cases a short &lt;summary&gt; is the best fix.

## Examples

### Violation

```cs
/// <summary>
/// Hosts nested values
/// </summary>
internal class Container
{
    private struct Token
    {
    }
}
```

### Correction

```cs
/// <summary>
/// Hosts nested values
/// </summary>
internal class Container
{
    /// <summary>
    /// Stores the lookup token used by the container
    /// </summary>
    private struct Token
    {
    }
}
```
