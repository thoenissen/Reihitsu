# RH0311 — goto-Statement should be preceded by a blank line

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH0311       |
| **Category** | Formatting   |
| **Severity** | Warning      |
| **Code Fix** | ✗            |

## Description

This rule checks that `goto` statements are preceded by a blank line. The rule does not apply when the `goto` statement is the first statement in a block.

## Why is this a problem?

Separating statements with blank lines improves code readability by visually grouping related logic. A blank line before a `goto` statement makes it clear that an unconditional jump is about to occur, drawing attention to the change in control flow.

## How to fix it

Add a blank line before the `goto` statement. The rule does not apply when the statement is the first statement in a block.

## Examples

### Violation

```cs
var status = GetStatus();
goto HandleStatus;
```

### Correction

```cs
var status = GetStatus();

goto HandleStatus;
```
