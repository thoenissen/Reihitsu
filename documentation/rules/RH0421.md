# RH0421 - Non-private methods must be documented

| Property | Value |
|----------|-------|
| **ID** | RH0421 |
| **Category** | Documentation |
| **Severity** | Warning |
| **Code Fix** | ❌ |

## Description

This rule requires non-private method declarations to have XML documentation. A declaration counts as documented when it has a &lt;summary&gt; tag or uses &lt;inheritdoc/&gt;.

## Why is this a problem?

Undocumented non-private method declarations make the code harder to understand because readers have to infer intent from implementation details instead of reading a clear contract.

## How to fix it

Add a meaningful XML comment above the declaration. In most cases a short &lt;summary&gt; is the best fix.

## Examples

### Violation

```cs
/// <summary>
/// Represents a worker
/// </summary>
internal class Worker
{
    internal void Execute()
    {
    }
}
```

### Correction

```cs
/// <summary>
/// Represents a worker
/// </summary>
internal class Worker
{
    /// <summary>
    /// Executes the pending work items
    /// </summary>
    internal void Execute()
    {
    }
}
```
