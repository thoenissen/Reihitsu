# RH0377 — Opening parenthesis must be on declaration line

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH0377       |
| **Category** | Formatting   |
| **Severity** | Warning      |
| **Code Fix** | ✓            |

## Description

This rule ensures that methods and constructors place the opening parenthesis on the declaration line.

## Why is this a problem?

Inconsistent formatting makes code harder to read and creates unnecessary noise in diffs.

## How to fix it

Adjust the affected formatting manually or apply the provided code fix.

## Examples

### Violation

```cs
internal class TestClass
{
    void Method
    (int value)
    {
    }
}
```

### Correction

```cs
internal class TestClass
{
    void Method(int value)
    {
    }
}
```
