# RH0109 — Debug.Fail must provide message text

| Property | Value |
|----------|-------|
| **ID** | RH0109 |
| **Category** | Design |
| **Severity** | Warning |
| **Code Fix** | ❌ |

## Description

Calls to `Debug.Fail` must provide meaningful message text.

## Why is this a problem?

`Debug.Fail(null)`, `Debug.Fail("")`, or `Debug.Fail("   ")` give no clue why the failure path was reached.

## How to fix it

Pass a descriptive non-empty, non-whitespace message to `Debug.Fail`.
