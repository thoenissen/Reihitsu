# RH0318 — unchecked-Statement should be preceded by a blank line

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH0318       |
| **Category** | Formatting   |
| **Severity** | Warning      |
| **Code Fix** | ✗            |

## Description

This rule checks that `unchecked` statements are preceded by a blank line. The rule does not apply when the `unchecked` statement is the first statement in a block.

## Why is this a problem?

Separating statements with blank lines improves code readability by visually grouping related logic. A blank line before an `unchecked` block makes it clear that overflow checking is being explicitly disabled for a section of code.

## How to fix it

Add a blank line before the `unchecked` statement. The rule does not apply when the statement is the first statement in a block.

## Examples

### Violation

```cs
var a = int.MaxValue;
unchecked
{
    var b = a + 1;
}
```

### Correction

```cs
var a = int.MaxValue;

unchecked
{
    var b = a + 1;
}
```
