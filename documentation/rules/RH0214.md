# RH0214 — Internal field names should be in PascalCase

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH0214       |
| **Category** | Naming       |
| **Severity** | Warning      |
| **Code Fix** | ✓            |

## Description

Checks that internal field names follow PascalCase naming conventions. Each word in the name should start with an uppercase letter, with no underscores or other separators.

## Why is this a problem?

PascalCase for internal fields maintains consistency with other non-private members. Inconsistent naming makes the codebase harder to read and navigate.

## How to fix it

Rename the internal field to use PascalCase.

## Examples

### Violation

```cs
internal int myField;
```

### Correction

```cs
internal int MyField;
```
