# RH0407 - Private interfaces must be documented

| Property | Value |
|----------|-------|
| **ID** | RH0407 |
| **Category** | Documentation |
| **Severity** | Warning |
| **Code Fix** | ❌ |

## Description

This rule requires private interface declarations to have XML documentation. A declaration counts as documented when it has a &lt;summary&gt; tag or uses &lt;inheritdoc/&gt;.

## Why is this a problem?

Undocumented private interface declarations make the code harder to understand because readers have to infer intent from implementation details instead of reading a clear contract.

## How to fix it

Add a meaningful XML comment above the declaration. In most cases a short &lt;summary&gt; is the best fix.

## Examples

### Violation

```cs
/// <summary>
/// Hosts nested contracts.
/// </summary>
internal class Container
{
    private interface IMessageSink
    {
    }
}
```

### Correction

```cs
/// <summary>
/// Hosts nested contracts.
/// </summary>
internal class Container
{
    /// <summary>
    /// Receives messages produced by the container
    /// </summary>
    private interface IMessageSink
    {
    }
}
```
