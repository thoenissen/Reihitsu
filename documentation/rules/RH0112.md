# RH0112 — File may only contain a single top-level type

| Property | Value |
|----------|-------|
| **ID** | RH0112 |
| **Category** | Design |
| **Severity** | Warning |
| **Code Fix** | ❌ |

## Description

A source file should contain at most one top-level class, struct, record, interface, or enum.

## Why is this a problem?

Multiple top-level types in one file make it harder to find the right declaration, review changes, and keep files focused on a single responsibility.

## How to fix it

Move each additional top-level type into its own file. Nested types inside another type are not affected by this rule.

## Examples

### Violation

```cs
namespace Example;

public class User
{
}

internal record UserDto(string Name, string Email);

internal interface IUserRepository
{
    User GetUser(int id);
}
```

### Correction

```cs
namespace Example;

public class User
{
}
```

```cs
namespace Example;

internal record UserDto(string Name, string Email);
```

```cs
namespace Example;

internal interface IUserRepository
{
    User GetUser(int id);
}
```
