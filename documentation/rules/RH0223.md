# RH0223 — Deconstruction variable names should be in camelCase

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH0223       |
| **Category** | Naming       |
| **Severity** | Warning      |
| **Code Fix** | ✓            |

## Description

Checks that deconstruction variable names follow camelCase naming conventions. The first word should start with a lowercase letter, and each subsequent word should start with an uppercase letter.

## Why is this a problem?

camelCase for local variables is a fundamental C# naming convention. Deconstruction variables are local variables and should follow the same naming rules for consistency.

## How to fix it

Rename the deconstruction variables to use camelCase.

## Examples

### Violation

```cs
var (Name, Age) = GetPerson();
```

### Correction

```cs
var (name, age) = GetPerson();
```
