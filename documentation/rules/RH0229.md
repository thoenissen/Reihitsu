# RH0229 — Record names should be in PascalCase

| Property | Value |
|----------|-------|
| **ID** | RH0229 |
| **Category** | Naming |
| **Severity** | Warning |
| **Code Fix** | ✓ |

## Description

Ensures `record` and `record class` type names use PascalCase.

## Why is this a problem?

Type names are central API surface. Using inconsistent casing for record types makes code harder to scan and breaks established C# naming expectations.

## How to fix it

Rename the record so each word starts with an uppercase letter.

## Examples

### Violation

```cs
public record orderrecord(string Value);
```

### Correction

```cs
public record Orderrecord(string Value);
```
