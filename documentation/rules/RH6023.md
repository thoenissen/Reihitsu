# RH6023 — Assignment operators must be spaced correctly

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH6023       |
| **Category** | Spacing |
| **Severity** | Warning      |
| **Code Fix** | ✓            |

## Description

An assignment operator must be separated from the token before it and the token after it by exactly one space. This covers `=`, the compound assignments (`+=`, `-=`, `*=`, `/=`, `%=`, `&=`, `|=`, `^=`, `<<=`, `>>=`, `>>>=`, `??=`) and the `=` used in initializers (`int x = 2`) and alias/member assignments (`using A = B`, `new { A = x }`).

Operators that wrap to a new line are left untouched.

## Why is this a problem?

Missing spaces (`x=2`) run the operands together and hurt readability. Extra spaces — for example padding used to vertically align the `=` of consecutive assignments — are fragile: they break the moment an identifier length changes, produce noisy diffs, and are inconsistent with the single-space spacing enforced everywhere else.

## How to fix it

Use a single space on both sides of the operator. The code fix applies this change automatically.

## Examples

### Violation

```cs
internal class TestClass
{
    void Method()
    {
        var a   = 2;
        var abc = 3;
    }
}
```

### Correction

```cs
internal class TestClass
{
    void Method()
    {
        var a = 2;
        var abc = 3;
    }
}
```
