# RH6023 — Code must not contain alignment padding

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH6023       |
| **Category** | Spacing |
| **Severity** | Warning      |
| **Code Fix** | ✓            |

## Description

There must be only a single space between two tokens on the same line. Runs of more than one consecutive space (or tab) used to vertically align tokens are not allowed.

Leading indentation, trailing whitespace, string and character literal content, interpolated string text, and comment or documentation text are not affected by this rule.

## Why is this a problem?

Alignment padding is fragile: it breaks the moment an identifier length changes, so a single rename forces every aligned line to be re-padded. It also produces noisy diffs and is inconsistent with the single-space spacing enforced everywhere else.

## How to fix it

Collapse the padding to a single space. The code fix applies this change automatically.

## Examples

### Violation

```cs
internal class TestClass
{
    void Method()
    {
        var a   = 2;
        var abc = 3;
    }
}
```

### Correction

```cs
internal class TestClass
{
    void Method()
    {
        var a = 2;
        var abc = 3;
    }
}
```
