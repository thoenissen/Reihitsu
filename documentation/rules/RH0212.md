# RH0212 — Private field names should be in _camelCase

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH0212       |
| **Category** | Naming       |
| **Severity** | Warning      |
| **Code Fix** | ✓            |

## Description

Checks that private field names follow _camelCase naming conventions with an underscore prefix. The first word after the underscore should start with a lowercase letter, and each subsequent word should start with an uppercase letter.

## Why is this a problem?

The underscore prefix convention for private fields clearly distinguishes them from local variables and parameters. Inconsistent naming makes it harder to understand the scope and lifetime of variables at a glance.

## How to fix it

Rename the private field to use _camelCase with an underscore prefix, including `private static readonly` fields.

## Examples

### Violation

```cs
private int count;
private int Count;
private static readonly int CacheLimit = 10;
```

### Correction

```cs
private int _count;
private static readonly int _cacheLimit = 10;
```
