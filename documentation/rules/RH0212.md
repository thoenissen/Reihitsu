# RH0212 — Private field names should be in _camelCase

| Property     | Value        |
|--------------|--------------|
| **ID**       | RH0212       |
| **Category** | Naming       |
| **Severity** | Warning      |
| **Code Fix** | ✓            |

## Description

Checks that private instance field names follow _camelCase naming conventions with an underscore prefix. The first word after the underscore should start with a lowercase letter, and each subsequent word should start with an uppercase letter. Private `static readonly` fields are excluded and should follow RH0228 instead.

## Why is this a problem?

The underscore prefix convention for private fields clearly distinguishes them from local variables and parameters. Inconsistent naming makes it harder to understand the scope and lifetime of variables at a glance.

## How to fix it

Rename the private field to use _camelCase with an underscore prefix.

## Examples

### Violation

```cs
private int count;
private int Count;
```

### Correction

```cs
private int _count;
```
