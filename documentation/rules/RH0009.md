# RH0009 — Use lambda syntax

| Property | Value |
|----------|-------|
| **ID** | RH0009 |
| **Category** | Clarity |
| **Severity** | Warning |
| **Code Fix** | ✓ |

## Description

This rule prefers lambda expressions over anonymous methods written with the `delegate` keyword.

## Why is this a problem?

Lambda expressions are the normal C# style for inline functions. They are shorter and usually easier to read than the older `delegate` syntax.

## How to fix it

Replace the anonymous method with an equivalent lambda expression.

## Examples

### Violation

```cs
Func<int, int> increment = delegate(int value) { return value + 1; };
```

### Correction

```cs
Func<int, int> increment = (int value) => value + 1;
```
