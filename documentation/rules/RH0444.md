# RH0444 - Single-line comments must not use documentation style slashes

| Property | Value |
|----------|-------|
| **ID** | RH0444 |
| **Category** | Documentation |
| **Severity** | Warning |
| **Code Fix** | ✓ |

## Description

This rule forbids regular single-line comments that start with `///`.

## Why is this a problem?

Three leading slashes are reserved for XML documentation comments. Using them for ordinary comments makes the code look documented when it is not, and can confuse editors and readers.

## How to fix it

Change ordinary comments to use `//` instead of `///`.

## Examples

### Violation

```cs
/// temporary workaround
DoWork();
```

### Correction

```cs
// temporary workaround
DoWork();
```
