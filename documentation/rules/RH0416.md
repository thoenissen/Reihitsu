# RH0416 - Non-private delegates must be documented

| Property | Value |
|----------|-------|
| **ID** | RH0416 |
| **Category** | Documentation |
| **Severity** | Warning |
| **Code Fix** | ❌ |

## Description

This rule requires non-private delegate declarations to have XML documentation. A declaration counts as documented when it has a &lt;summary&gt; tag or uses &lt;inheritdoc/&gt;.

## Why is this a problem?

Undocumented non-private delegate declarations make the code harder to understand because readers have to infer intent from implementation details instead of reading a clear contract.

## How to fix it

Add a meaningful XML comment above the declaration. In most cases a short &lt;summary&gt; is the best fix.

## Examples

### Violation

```cs
internal delegate void ProgressHandler();
```

### Correction

```cs
/// <summary>
/// Handles progress notifications from the import pipeline
/// </summary>
internal delegate void ProgressHandler();
```
