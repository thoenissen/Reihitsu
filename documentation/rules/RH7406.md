# RH7406 — Non-override properties should not be placed in override regions

| Property | Value |
|----------|-------|
| **ID** | RH7406 |
| **Category** | Organization |
| **Severity** | Warning |
| **Code Fix** | ❌ |

## Description

This rule forbids non-override properties from being placed inside a top-level override region whose label is used by override properties in the same type.

## Why is this a problem?

Override regions are meant to group inherited members by the type that originally introduced them. Placing ordinary properties inside those regions makes it harder to distinguish override surface area from type-specific state or convenience members.

## How to fix it

Move each non-override property out of the override region and into a normal organizational region such as `Properties`, or leave it ungrouped if that matches the surrounding style.

## Examples

### Violation

```cs
internal abstract class BaseProcessor
{
    public virtual string Name => string.Empty;
}

internal sealed class ReportProcessor : BaseProcessor
{
    #region BaseProcessor

    public override string Name => base.Name;
    public string DisplayName => Name;

    #endregion // BaseProcessor
}
```

### Correction

```cs
internal abstract class BaseProcessor
{
    public virtual string Name => string.Empty;
}

internal sealed class ReportProcessor : BaseProcessor
{
    #region BaseProcessor

    public override string Name => base.Name;

    #endregion // BaseProcessor

    #region Properties

    public string DisplayName => Name;

    #endregion // Properties
}
```
