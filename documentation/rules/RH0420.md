# RH0420 - Destructors must be documented

| Property | Value |
|----------|-------|
| **ID** | RH0420 |
| **Category** | Documentation |
| **Severity** | Warning |
| **Code Fix** | ❌ |

## Description

This rule requires destructor declarations to have XML documentation.

## Why is this a problem?

Destructors are uncommon and usually indicate important cleanup behavior. Without documentation, that behavior is easy to miss.

## How to fix it

Add a meaningful XML comment above the declaration. In most cases a short &lt;summary&gt; is the best fix.

## Examples

### Violation

```cs
/// <summary>
/// Manages unmanaged resources
/// </summary>
internal class ResourceOwner
{
    ~ResourceOwner()
    {
    }
}
```

### Correction

```cs
/// <summary>
/// Manages unmanaged resources
/// </summary>
internal class ResourceOwner
{
    /// <summary>
    /// Releases unmanaged resources that were not disposed explicitly
    /// </summary>
    ~ResourceOwner()
    {
    }
}
```
