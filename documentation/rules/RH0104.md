# RH0104 — Access modifier must be declared

| Property | Value |
|----------|-------|
| **ID** | RH0104 |
| **Category** | Design |
| **Severity** | Warning |
| **Code Fix** | ✓ |

## Description

Every type and member that supports accessibility should declare it explicitly. Depending on language defaults makes code harder to read because the reader has to infer visibility from context.

Partial methods without an explicit accessibility modifier are exempt because that form is valid C# syntax and intentionally relies on the language default.

## Why is this a problem?

Implicit accessibility hides intent. Explicit modifiers make the public surface and implementation details obvious at a glance.

## How to fix it

Add the default accessibility modifier for the declaration context, such as `internal` for top-level types and `private` for members.
