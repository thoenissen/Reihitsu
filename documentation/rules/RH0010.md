# RH0010 — Use readable conditions

| Property | Value |
|----------|-------|
| **ID** | RH0010 |
| **Category** | Clarity |
| **Severity** | Warning |
| **Code Fix** | ✓ |

## Description

This rule disallows Yoda conditions, where a constant is written on the left side of a comparison and the variable or expression is written on the right.

## Why is this a problem?

Conditions are easier to read when the changing value appears first. Writing the constant first makes the expression feel unnatural and slows down scanning.

## How to fix it

Swap the operands so the variable or expression is on the left side.

## Examples

### Violation

```cs
if (0 < count)
{
}
```

### Correction

```cs
if (count > 0)
{
}
```
