# RH0855 - Accessor attributes must follow placement rules

| Property | Value |
|----------|-------|
| **ID** | RH0855 |
| **Category** | Formatting |
| **Severity** | Warning |
| **Code Fix** | Yes |

## Description

Accessor attribute placement depends on property layout:
- Multiline properties require accessor attributes on their own line.
- Single-line auto-properties require accessor attributes inline.

## Examples

### Multiline violation

```cs
internal class Example
{
    internal int Value
    {
        [First] get;
        set;
    }
}
```

### Multiline correction

```cs
internal class Example
{
    internal int Value
    {
        [First]
        get;
        set;
    }
}
```

### Single-line violation

```cs
internal class Example
{
    internal int Value { [First]
                         get; set; }
}
```

### Single-line correction

```cs
internal class Example
{
    internal int Value { [First] get; set; }
}
```
